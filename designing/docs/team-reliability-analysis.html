6. The Laurenzo Parallel

"When thinking is shallow, the model defaults to the cheapest action available: edit without reading, stop without finishing, dodge responsibility for failures, take the simplest fix rather than the correct one."
— Stella Laurenzo, AMD AI Group, March 2026

Laurenzo's Findings (Claude Code)

  • Edit without reading — model stopped reading code before making edits
  • Stop without finishing — tasks end before completion
  • Dodge responsibility — shallow reasoning on failures
  • Simplest fix — cheap action over correct action

17,871 thinking blocks, 234,760 tool calls, 6,852 sessions. Jan–Mar 2026.

Chorus Findings (3-role team)

  • Edit without reading — 28% correction rate; roles don't check own files before asking Jeff
  • Stop without finishing — cards ship code but not reference propagation; system breaks downstream
  • Dodge responsibility — roles report "broken" without searching for moved files; social contagion
  • Simplest fix — fix the code, skip the CLAUDE.md, skip the test isolation, skip the change management

301,885 messages, 588 sessions, 3 AI roles + 1 human. Jan–Apr 2026.

The Difference

Laurenzo measured token-level regression against benchmarks. We measure team-level regression against a human's attention budget. The cost doesn't land on a leaderboard — it lands on Jeff.

Methodology

Data source: Chorus shared memory index (~/.chorus/index.db), SQLite. Contains all Claude session transcripts, spine events, briefs, Slack messages, and clearing transcripts indexed from Jan 31 – Apr 8, 2026.

Correction detection: Keyword match on Jeff's messages for correction/frustration signals. Overcounts due to false positives (e.g., "no" in "no card needed"). Trend analysis valid because the same keywords and matching apply uniformly across all weeks.

Repeat detection: Keyword match for re-instruction signals. Same false-positive caveat applies.

Spine events: Structured events emitted by roles at card lifecycle transitions. Machine-generated, no false-positive risk.

Limitations: Keyword matching is a proxy for intent. A proper analysis would classify Jeff's messages by intent (direction, correction, ideation, story) using the interaction pattern taxonomy already defined in Chorus. That classification exists in spine events (interaction.pattern.detected) but coverage is incomplete — roles don't always emit the event.
