Strategic Insights

1. The Clearing is your Genesis bet. Multi-party AI chat with grounded roles is novel. Nobody else has this running with honest system prompts, context injection, and soon tool access. It's also the highest risk — Haiku quality, hallucination, cost. The evolution path: Genesis (today) → Custom (with tooling #176) → Product (if others can use it).
2. The Workflow Engine is your moat. Auto-manifest creation, brief routing, cycle time measurement, piece flow visualization — this is where the coordination intelligence lives. It's custom-built and moving toward product. If you open-source Chorus, this is the component that's hardest to replicate because it encodes your operating model.
3. CLAUDE.md is evolving fast. From static rules file → generated fragments → auto-versioned (Werk) → spine architecture. This is moving rightward faster than anything else on the map. The 209-rules-18-enforced gap means it's at an inflection point — either it becomes a real product component or it collapses under its own weight.
4. The platform layer is all commodity. SQLite, filesystem, Docker, launchd, Claude API — nothing custom here. This is correct. Your value is in the coordination and interaction layers, not in infrastructure choices. If someone cloned your platform stack, they'd have nothing. If they cloned your workflow engine + Clearing + brief system, they'd have something.
5. The Chorus Index is under-leveraged. 23.6k records, ambient indexing, FTS5 search — but it's only used for manual /chorus search. It should feed The Clearing (context injection), the defect poller (pattern correlation), and the workflow engine (decision traceability). It's the memory that makes everything else smarter.
6. Jeff's position on the map. Jeff sits at the very top — the only user need. Everything flows from his direction. The strategic question isn't "what do we build next" — it's "how do we reduce the distance between Jeff's intent and system response." Every component that shortens that distance (Board CLI, voice capture, Clearing) is high-value. Every component that adds steps (manual briefs, CLAUDE.md editing) is friction to eliminate.

Where to Invest Next

Evolve right (reduce custom): CLAUDE.md → spine (already in motion), Session hooks → more declarative, Board CLI → richer (product labels shipped tonight)
Invest in Genesis (novel value): Clearing tool access (#176), SWAT pre-flight (#194), Chorus Index as context source for everything, ops-agent headless monitoring
Don't touch (commodity, working): Filesystem briefs, SQLite FTS5, Docker, launchd — these are solved. Swapping them adds risk for zero user value.
