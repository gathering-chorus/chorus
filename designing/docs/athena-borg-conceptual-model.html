The red link is the impedance: framework.ttl declares borg:Environment owl:equivalentClass chorus:Service (and framework:Service) — one concept wearing three names across three namespaces. That's what the composed core dissolves: one chorus:Service, products extend it, no equivalentClass stitching.

What it tells us

Borg has 12 domains but ~8 classes, they don't map 1:1, five domains have no class, three design-doc classes were never written into borg.ttl, and the core concept (Service/Environment) has three names bridged by owl:equivalentClass. This is exactly the impedance the composed chorus-core.ttl removes — and it's why "OWL defines the API" can't work until the names are one set: a generated API off this would emit three names for one thing.

Baseline (Jeff, 2026-06-02): Athena v2 is canonical. The data/athena/tree.json model — its 33 domain names + the four axes (domain · ownedBy · atStep · shape) — is the single source of truth. The OWL (chorus.ttl / borg.ttl / framework.ttl) reconciles to v2, never the reverse.

So the fix is one-directional: one composed OWL with chorus:Service as the single concept (retire the borg:Environment ≡ equivalentClass bridge); borg's Engine/Environment/Resource/Monitor/AlertRule become subClassOf chorus:Facet/Service under the core; every class re-homed under a v2 domain name; the five class-less domains and three doc-only classes resolved against v2. Part of Silas's chorus.ttl→core refactor (#3185) + the self-model framework.
