The Chorus Spine

Value stream architecture — how decisions flow through three thirds
Jeff — The Pull Signal

Jeff directs. He moves cards to Now, makes decisions, and observes flow.
He is not the relay between roles — the spine is.
"Pretty soon I could see my job being mostly moving cards to Now and watching them flow."

🪶 Wren — Upper Third

Capturing · Directing · Designing

vertebrae: capture → shape → prioritize → manifest → handoff

The product mind. Every piece of work enters through here — captured as a seed, shaped into a card, given priority and direction. Nothing reaches the building zone without a manifest.

Card-First Gate

No work without a card. No card without work ahead of it. Session audits enforce: stale detection, retroactive card warnings, snapshot diffs.

Manifest-First Gate

Every card in Now must have a workflow manifest before work starts. The manifest is the handshake between Wren's third and Kade's — no manifest, no handoff.

⚡ Kade — Middle Third

Building

vertebrae: receive brief → build → test → commit → advance workflow

The engineering hands. Receives shaped work with clear direction, builds it, proves it with tests, and advances the workflow to review. Ships fast because the input is clean.

Collection Engine

Pod-backed CRUD pattern: interface → service → handler → views → routes → navbar → mind map node. Shipped: music, photos, books, notes, watchlist, reading, cooking, todo, social posts, blog, gallery, property, ideas.
SHIPPED

Sitemap Scraper

Playwright-driven visual health check. Screenshots every page, detects broken images, raw UUIDs, empty pages. HTML report with issue summary. The automated gemba walk.
SHIPPED

Build Cycle SLAs

Instrument time-to-build, time-to-test, time-to-deploy. The feedback loop that keeps the middle third fast.
PLANNED

Navigation Consistency

Navbar menu and mind map graph are two navigation trees into the same data. Both must resolve, both must agree. E2E tests or deprecate one.

🔧 Silas — Lower Third

Proving

vertebrae: review → verify → deploy → monitor → alert

The operational foundation. Reviews what was built, verifies it works in production, monitors it running, and alerts when something breaks. The spine can't flow if the bottom isn't solid.

Infrastructure Guardrails

PreToolUse hooks block dangerous operations (docker rm, kill PIDs, force push). Read-only escape hatch for diagnostics. The immune system of the spine.
SHIPPED

Defect Escape Hatch

Proactive log polling → auto-ticketing. Loki queries surface errors before Jeff notices them. False positive filtering (in progress — bash array bug found).
SHIPPED

macOS Native Alerts

Alertmanager → Python webhook receiver → osascript notifications. Banners for warnings, modal dialogs for critical. Storm protection (5/batch). Replaced Slack receivers.
SHIPPED

LAN Smoke Tests

Post-deploy: hit the app from LAN IP, verify health + login page renders. Catches networking regressions that localhost tests miss. Wired into start/restart/deploy.
SHIPPED

Headless Ops Agent

claude -p for autonomous monitoring. Read-only Phase 1: health checks, log triage, disk monitoring. No write operations without approval gates.

How a piece of work flows
📱

1. Jeff texts himself an idea

SMS → Twilio → capture triage → seed. Raw material enters the system. Or Jeff moves a card to Now directly.

↓
🪶

2. Wren shapes it

Seed becomes a card. Card gets priority, description, owner. Product direction attached. Moved to Now when Jeff pulls it.

↓
🚧
GATE: Card must exist + manifest must exist before crossing into Building
↓
📋

3. Workflow manifest created

Auto-trigger on move to Now. 2-step workflow: Kade builds (step 1), Silas reviews (step 2). Brief auto-routed to engineer/briefs/.

↓
⚡

4. Kade builds

Reads brief on session start. Builds the feature, writes tests, commits. Advances workflow when done. Brief auto-routed to architect/briefs/.

↓
🚧
GATE: Tests pass + workflow advanced before crossing into Proving
↓
🔧

5. Silas reviews and verifies

Code review, security check, deployment verification. Advances workflow → card moves to Done. Monitoring continues.

↓
✅

6. Done — visible on Value Stream

Workflow complete. Cycle time measured. Visible on /value page. Jeff sees it flow through without relaying a single message.
