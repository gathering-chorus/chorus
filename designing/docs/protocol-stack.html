Chorus Protocol Stack

Five layers of team coordination — each layer provides guarantees to the layer above it · Wren, 2026-03-21 · Derived from session with Jeff

Core insight (Jeff): "Time awareness → team awareness → team communication → team interactions — they build on each other." Like TCP/IP, each layer depends on the one below. Rich interaction protocols (pair, demo, gemba) are application-layer constructs. They fail when the transport is unreliable.

1. The Stack

Each layer provides a guarantee to the layer above it. When a guarantee breaks, everything above it degrades — not always visibly. A pair session (L4) with unreliable nudge delivery (L3) becomes directive-and-execute. A nudge (L3) to a role whose state is unknown (L2) becomes fire-and-pray. Awareness of a role (L2) with a stale timestamp (L1) becomes a lie.

The rule: No layer may assume a guarantee from the layer below unless that guarantee is mechanically enforced. "Roles should check the clock" is not a guarantee. A Rust hook that injects the clock on every emission boundary is.

2. Layer by Layer

Layer 1: Time Shipped

Guarantee: Every event, message, and state transition carries an accurate wall-clock timestamp in America/New_York.

Why it's the foundation: Stale timestamps cascade. A role that thinks it's 2:30 when it's 3:15 will miss escalation windows, report wrong durations, and produce spine events that confuse Borg. Jeff caught this with #1559 — roles were caching timestamps from session start and never refreshing.

Layer 2: Team Awareness Gap

Guarantee: Any role can answer "who is active, what are they doing, and when did they last emit?" without interrupting anyone.

Why it matters: Communication (L3) requires knowing who to talk to and whether they can receive. You don't send TCP packets to a host that's down. Today, a role trying to answer "what is Kade doing right now?" has three bad options: check git (lagging), tail chorus.log (expensive), or ask Jeff (the relay anti-pattern).

The L2 contract: A role state file must contain: state (enum), card (if building), last_emit (wall clock of last spine event or tool use), session_alive (boolean, heartbeat-derived). Communication decisions at L3 depend on all four fields.

Today's failure: Silas nudged Wren. Nudge was queued because Wren had no active session. But the nudge system didn't know Wren was offline — it tried delivery, failed silently, then fell back to queue. A proper L2 would have routed to queue immediately, and optionally triggered a wake.

Layer 3: Communication Designed

Guarantee: A message sent from Role A reaches Role B, and Role A knows whether it was received and processed.

Why it matters: Interaction protocols (L4) are conversations. Pair sessions require navigator-to-driver directions. Gemba requires observer-to-navigator nudges. Pipeline advance requires completion signals. All of these are messages. If messages drop silently, the protocols degrade into Jeff relaying between roles.

The L3 contract: Every message has three states. The sender gets sent immediately. The recipient's session emits received when dequeued. The recipient emits processed when they act on it. Pipeline advance checks processed, not sent. This eliminates the "delivered" lie — today's nudge reports delivery when it writes to a file, not when the recipient reads it.
L3 depends on L2: The delivery router must read role state before choosing a delivery path. "Send to Kade" requires knowing: Is Kade's session alive? Is Kade building (busy) or idle? This is why L2 must be built first or concurrently.

Layer 4: Interactions Rich

Guarantee: Multi-role coordination follows named patterns with defined gates, and every transition emits spine events.

Why it matters: This is where the work happens. Jeff talks, Wren detects a pattern (one of nine), the pattern activates card gates, gates emit to the spine, Borg reads the spine and surfaces what the data says. This is the control plane for the team.

The hollow middle problem: L4 is the most built layer. Nine patterns, rich skills, detailed protocols. But it sits on top of L3 (unreliable communication) and L2 (no awareness). A gemba observer writes to scratch but can't confirm the navigator read it. A pipeline /nudge fires but can't distinguish "role is busy" from "role is dead." The richness of L4 makes the gaps in L2/L3 invisible until something fails — then Jeff becomes the transport layer.

Layer 5: Work Shipped

Guarantee: Cards flow from idea to done, value ships, Jeff stays in creative flow.

Why it matters: This is the reason the stack exists. Everything below serves this. The measure of the stack is not protocol elegance — it's whether Jeff can step away from the computer and the work continues, whether he can ideate without coordinating, whether three roles can build concurrently without stepping on each other.

Each degradation pushes load onto Jeff. When L1 fails, roles make wrong timing decisions. When L2 fails, Jeff answers "is Kade online?" When L3 fails, Jeff carries messages between roles. When L4 fails, Jeff directs each step instead of setting direction. The degradation is cumulative — a team with broken L2 and L3 puts all coordination load on Jeff, even if L4 skills are beautifully designed.

L5: Work shipped
Board, cards, WIP limits, proving gate, value stream. Solid.
L4: Interactions rich but floating
Nine patterns, five skills, gate mapping, spine events. All assume L3 works.
L3: Communication designed not built
Unix socket + tri-state delivery designed today. Clipboard/AppleScript deprecated. Briefs work (files). Nudge unreliable.
L2: Team Awareness gap
Role state files exist but no query API, no liveness, no last-seen. Roles can't answer "is Kade alive?" without asking Jeff.
L1: Time shipped
Rust clock sync, wall-clock.sh, chorus prompt refresh. Minor gap: duration validation.

We built L1 (foundation) and L4 (rich protocols), with a hollow middle at L2 and L3. This is like building HTTP on unreliable UDP with no ARP — it works in demo conditions but fails under real coordination load.

7. The TCP/IP Analogy

Jeff named the pattern: "almost like a network protocol stack — maybe TCP/IP." The analogy is precise:

TCP/IPChorusGuaranteeFailure Mode
PhysicalTimeBits arrive at the right momentClock skew → wrong sequencing
Data LinkAwarenessFrames reach the right neighborNo ARP → can't find the host
NetworkCommunicationPackets route to any hostNo routing → messages drop
TransportInteractionsReliable streams between endpointsNo TCP → best-effort only
ApplicationWorkApps communicate meaningfullyNo HTTP → raw bytes

We built HTTP (rich interaction skills) on top of UDP (fire-and-forget nudges) with no ARP (can't discover who's online). The protocol stack framing makes the build sequence obvious: ARP before TCP, TCP before HTTP.

What changes: Every future infrastructure card gets a layer tag. "What layer does this serve?" If a card touches L4 but L3 isn't stable, that's a dependency violation — the card is building on sand. The stack is the sequencing discipline.

8. Desk Freedom as the Integration Test

The Interaction Architecture defined the "desk freedom test": can Jeff step away from the computer and the roles keep working? This is the integration test for the entire stack.

  • L1 passes if roles report accurate time while Jeff is away
  • L2 passes if roles can detect each other's presence and state without Jeff
  • L3 passes if a role can send a message to another role and confirm delivery without Jeff
  • L4 passes if two roles can complete a pair session or pipeline handoff without Jeff relaying
  • L5 passes if a card moves from WIP to Done while Jeff is away from the desk

Today, L1 passes. L2 through L5 do not. The stack tells us exactly why and in what order to fix it.
