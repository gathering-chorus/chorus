Convergence — Value-Stream Design — the boundary; bringing the outside world into the model

Wren · 2026-06-21 · INSTANCE of value-stream-template.html · sibling of athena / borg / loom / clearing / werk · owner Kade · DRAFT 0.1 (verb-family ideation, card #3545)
Scope. This is Convergence’s value stream — one instance of the api-driven value-stream template. Convergence is the boundary: it takes data from heterogeneous outside sources (an existing system, a legacy schema, an external feed) and brings it into the model, reconciled and conformed. Where Athena projects the model outward (generate), Convergence pulls the world inward (integrate). It is the least-drafted of the six streams — this 0.1 names the shape from first principles + the integration literature.
Convergence is a LINE, and it’s the inbound twin of Athena. Three verbs: ingest → map → land, event-triggered by incoming data, produces-and-terminates per batch. The symmetry: Athena = inward generation (model → surfaces); Convergence = boundary ingestion (sources → model). They share the same membrane — the OWL model / canonical form — approached from opposite sides. Convergence is also the natural home of Borg’s assimilation mode: the herald/xray discovers an existing system’s structure, and Convergence is how that discovered data lands in the graph.
⚠ As-is vs to-be. Convergence exists today mostly as cards and intent (#3392 ICD/convergence TTLs → chorus tree + Fuseki load-path; #2291 Chorus-extraction-from-Gathering), not a built pipeline. The ICD (Interface Control Document) gate + harvesters are the closest running pieces. This 0.1 is a greenfield design — the integration patterns below are the to-be the verbs instance, not current behavior.

1 · The convergence-* cycle

An integration pipeline collapsed to its three essentials. Each verb is one well-trodden body of integration practice (§6).

VerbWhat it doesFramework anchorOwner
ingestPull data from heterogeneous sources as tracked units, with provenance — each source has its own schema, keys, formats.NiFi / Flow-Based Programming (FlowFiles over a graph)Kade
mapRoute by content and transform toward the canonical model — bridge incompatible formats. The ETL-vs-ELT choice of where transform runs.EIP Message-Translator + Content-Based RouterKade
land ⭐Reconcile-before-load: schema-map, cleanse, dedup, entity-resolve to a golden master — THEN write into the system of record. Source data never matches; you reconcile first.record linkage · canonical data modelKade

The step it recurses under is open (likely a Harvesting / boundary step, or cross-stream — see the fork). Orchestrated by a chorus_convergence loop (the chorus_werk / chorus_athena analog), one trigger per inbound batch.

The load-bearing verb is land, and its heart is the RECONCILE, not the write. Jeff’s thirty-year integration lesson: “always reconcile before loading — source data never matches cleanly.” A land that skips reconciliation is the classic integration mistake — it loads dirty, duplicated, unconformed data and corrupts the model it was supposed to enrich. convergence-land is parse-external → reconcile-to-the-schema → write-conformant-instances. (This is the same data-plane discipline as athena-load, pointed at the boundary instead of at our own designs.)

2 · The engine — sources → canonical model

Convergence’s heart is a flow of tracked data units through a transform graph into a reconciled system of record. Data arrives as explicit packets (not transient variables), each carrying provenance; it is routed and mapped toward one canonical form; before it lands it is reconciled against what’s already there.

flowchart LR
  SRC["heterogeneous sources
(legacy schema, feed, discovered system)"] --> ING["ingest — flow-based intake (FlowFiles + provenance)"]
  ING --> MAP["map — route + translate toward canonical model"]
  MAP --> REC["reconcile — schema-map · cleanse · dedup · entity-resolve"]
  REC --> LAND["land — write conformant instances"]
  LAND --> SOR[("system of record — the OWL graph / canonical model")]
  REC -.golden master + source→row mapping.-> SOR
convergence — ingest → map → (reconcile) → land. The reconcile gate is where source data is conformed to the canonical model before it’s allowed into the system of record. Provenance is carried end-to-end.
  • ETL vs ELT is a map decision: transform-before-land (ETL, governance-first, fixed legacy sources — our likely default given the OWL/SHACL floor) vs land-raw-then-transform (ELT, elastic compute). The canonical-model conformance is non-negotiable; where it runs is the choice.
  • Provenance is first-class: every unit retraces to its source (NiFi Data Provenance) — essential for a system that assimilates existing systems and must explain where each fact came from.
  • Reconcile = the EIP Normalizer + record linkage: conform formats (ISO codes, canonical keys), cleanse, dedup, and resolve same-entity records into one golden master plus a mapping table of which sources fed which row.

5 · Sequencing — contract first, reconcile always

  1. Author the value-stream spine — value-stream-convergence + ingest/map/land as ValueStreamSteps. Land it (greenfield).
  2. ICD-first (DEC-095) — no harvester without a matching ICD provider section. The contract precedes the code; icd is the write-door floor.
  3. One source end-to-end — pick one inbound domain (e.g. #3392 ICD/convergence TTLs), run ingest→map→reconcile→land fully, proving the reconcile gate. The repetition is the rest of Convergence.
  4. Golden-master + provenance — entity resolution to one canonical record + a source→row mapping, so every landed fact retraces to its origin.
  5. Generate the mappings where possible — the source→canonical transform is itself a model artifact (the athena-make path applied to integration), so mappings don’t drift from the schema.

6 · Operating-model frames

  • Flow-Based Programming (J. Paul Morrison, 1970s) — ingest. Apps as networks of black-box processes passing discrete Information Packets over defined connections. = data as tracked units, not variables.
  • Apache NiFi — the operational ingest. FlowFiles (payload + attributes) through Processors over backpressured Connections, with first-class Data Provenance. = the ingest substrate.
  • Enterprise Integration Patterns (Hohpe & Woolf, 2003) — map. Message Translator, Content-Based Router, Normalizer, Canonical Data Model — 65 patterns along a message’s journey. = convergence-map.
  • ETL/ELT + Record Linkage — land. Transform placement (ETL vs ELT) + reconcile-before-load: schema mapping, cleansing, dedup, entity resolution to a golden master. = convergence-land.
