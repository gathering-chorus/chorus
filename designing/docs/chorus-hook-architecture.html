Current vs Proposed Sequence

Proposed: 6 phases — Session, Safety, Knowledge, Enrichment, Workflow, Observation. Within each phase, defined order. State flows forward.

Invariants

  1. Safety guards fire before any other hook
  2. Context enrichment fires exactly once per prompt cycle
  3. Gates never block observation hooks
  4. Card state verified against Vikunja SQLite, not API cache
  5. Acceptance events record who invoked them
  6. No hook creates production data
  7. Jeff's input is never corrupted by nudge injection
