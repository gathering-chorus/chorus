Service Design — Pulse

DRAFT • Owner: Wren (design) / Silas (implementation) • Domain: observability • From Jeff's metaphor (2026-03-31)

1. Experience

I glance at the Clearing and see the system's heartbeat. Cards moving, roles building, alerts firing — not because I asked, but because the system circulates. Like a pulse. I check it the way you check a pulse — am I alive? Is anything stuck? Is something spiking?

Pulse is the name for all system circulation — both the steady heartbeat signals (periodic health checks, daily reviews) and the nerve impulse signals (card pulled, seed arrived, alert fired). It's the short-term memory of the system, visible on the Clearing, searchable in Chorus.

4. What Is NOT Pulse

Not PulseWhyWhat it is instead
Chorus search queriesMemory retrieval, not circulationLong-term memory
File edits, git commitsWork output, not system signalBuild artifacts
Jeff's typingInput, not system stateInteraction
Tool call logs (Read, Glob, Grep)Too noisy, no signalDebug telemetry
Nudge contentThat's conversation, not state changeMessaging
The rule: Pulse events answer "what state did the system just enter?" not "what action did someone perform?" A card being pulled changes system state. A file being read does not.

Jeff's formulation (2026-03-31): "Clearing events via team-scan = short term memory. Chorus search = long term memory." The medium-term bridge layer is what was broken today — card events weren't flowing to the Clearing. #1887 fixed it.

6. Signal Hierarchy

Not all signals deserve the same delivery urgency:

UrgencyDeliveryExamples
CriticalNudge (osascript inject)Circuit break, service down, deploy failure, alert firing
ImportantBridge ambient eventCard pulled, card accepted, demo started, role state change
InformationalChorus log onlySeed received, pattern detected, hook allowed, heartbeat OK
The focus problem (2026-03-31): Jeff: "Every time you fire off a nudge while I am typing you steal my focus." Nudge delivery (osascript) steals Terminal focus. Critical alerts justify this. Normal card moves do not. Signal hierarchy prevents attention theft.

9. Open Questions (for team review)

Q1Should team-scan show card events (short-term) or just role activity? Currently it shows commands and file edits but not "Silas pulled #1884."
Q2Should there be a pulse API endpoint? GET /api/chorus/pulse returning recent events by urgency level — critical, important, informational.
Q3Heartbeat missed-detection: who monitors the monitors? If the ops agent itself goes down, who notices?
Q4Should the Clearing show a literal pulse visualization — a heartbeat line that flatlines when no events flow? Jeff checks it like checking a patient's vitals.
Q5How does pulse relate to the Borg? Borg observes patterns across sessions. Pulse is the raw signal Borg reasons over. Is Borg the brain interpreting the pulse?
