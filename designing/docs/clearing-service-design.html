Overview

The Clearing is a browser-based real-time surface for the Gathering team. Express + Socket.IO server on port 3470, served as a LaunchAgent. It aggregates role state, message streams, session tailing, alerts, and multi-party chat into a single view Jeff can glance at from any device — including his phone over Cloudflare tunnel.

Six components: Role Tiles, Message Stream, Session Tailer, Alerts Panel, Multi-Party Chat, Remote Access. Backed by six domains.

Message Stream REAL

Classified, visibility-filtered message feed. Whitelist-only — if a message type isn't explicitly matched, it's hidden. Jeff sees clean content, not protocol noise.

Source router.ts — MessageRouter class
8 message types: jeff-input, role-response, demo-ready, accept-request, blocked, role-to-role, system-error, pm-thinking
Classification Jeff input: always visible, spine metadata stripped
Role-to-role nudges: hidden
Demos, blocks, decisions: always visible
PM thinking: visible unless tool call
Default: hidden (whitelist approach)
Filtering Dedup: last 10 messages, @mention-stripped fuzzy match
System noise: XML tags, file paths, hook output
Bridge echo: [bridge] prefixed messages hidden
Max 200 messages retained
Gap: Bridge notifications inject into Jeff's terminal even when hidden on the Clearing (#2284). Jeff's own actions echo back. Needs audience routing in bridge-subscriber.js.
