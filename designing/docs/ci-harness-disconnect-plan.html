CI Harness Disconnect coordinated plan

Drafted 2026-04-27 · Silas (CI substrate) + Kade (tests substrate) · v0 awaiting Kade sign-off
The harness brief names the destination. This plan names the staging between two parallel tracks — CI substrate (Silas) and tests substrate (Kade) — so we don't discover gaps at integration. Each phase declares what each track produces, what crosses the boundary, and what has to be true before the next phase starts.

Why this exists

Jeff split the CI/test-substrate work between two roles after the harness brief surfaced (2026-04-27). Two parallel tracks with implicit interfaces is the canonical recipe for end-stage integration surprises. The fix is to make the interfaces explicit at each phase boundary and name the integration checkpoints up front, not on the day of the wire-up.

The interface that matters. Phase 2 introduces a single load-bearing API: a test runner that the CI workflow invokes by domain + category + budget. Get that contract right and the two tracks compose cleanly; get it wrong and we re-do Phase 2 twice. Phases 0 and 1 are independent enough that misalignment costs little; Phase 2 is the integration point.

The Phase 2 contract (the load-bearing interface)

PropertySpec
Binary locationplatform/scripts/run-tests (no extension, matches the nudge / cards / role-state convention). Thin wrapper that dispatches to per-language runners (jest, cargo nextest). Not in chorus-sdk (avoids a new build target); not folded into cards CLI (avoids scope creep on board ops).
Invocationchorus run-tests --domain=<subdomain-id> --category=hermetic,contract --budget=<seconds>
Domain resolutionGraph wins iff discover-tests ran within 24h and the subdomain has non-empty hasTest mappings. Otherwise filepath heuristic. Stale-graph-beats-fresh-filepath is worse than empty graph — the freshness gate prevents silent wrong-routing when #2515 alias drift sets in.
CategoryComma-separated list (union). Values: hermetic | integration | contract | smoke | perf | mutation. Filename-suffix-driven (Phase 0 convention). One invocation can request hermetic,contract for PR-required, or integration,smoke for nightly drift lane.
BudgetHard timeout in seconds. Runner kills the suite at budget; sets partial:true in output and exit code 2 (see below).
Exit code0 = all ran + all pass; 1 = all ran + any fail; 2 = budget exceeded (partial run); 3 = invocation error (bad domain, bad category, runner crash)
StdoutStructured JSON: 
{
  pass: N,
  fail: N,
  runtime_ms: N,
  partial: bool,                          // true iff exit 2
  failing_tests: [{name, file, error}],   // structured, not strings
  tests_unrun: [{name, file}],            // present iff partial:true
  hermeticity_tags: {...}
}
CI consumers can distinguish "these failed" from "these never ran" — required for honest reporting on budget timeout.
StderrHuman-readable progress + failures. Suppressed in CI by default; surfaced on non-zero exit.
Phase 2 exit:
  • One verification PR proves the round-trip: PR touches a single domain → CI invokes chorus run-tests --domain=X --category=hermetic → green merge.
  • One nightly run produces ≥1 card from a real signal (drift lane lights up).
  • Deploy smoke fires <30s post-reload; auto-rollback verified on synthetic failure.

Risks & gaps to watch

RiskWhy it bitesMitigation
Phase 0 categorization happens before #2515 lands; tests get tagged hermetic in filenames that turn out to be non-hermetic on audit. The audit and the categorization are the same operation. If Silas's audit and Kade's #2515 don't share a hermeticity definition, we tag inconsistently. Both tracks reference the same hermeticity rules (no network / no fs outside tmp / no time / no env-coupling / order-independent). Silas writes the rules into the DEC; Kade's #2515 cites them.
Phase 2 contract drift — Kade ships #2118 with one shape, Silas wires CI assuming another. The runner contract is the single integration point. Misread = Phase 2 redo. Contract is in this doc (Phase 2 table). Both tracks sign off before Kade starts implementation. Any change to the contract triggers a joint review.
Graph-vs-code-resident SPECIAL_ALIASES drift — Phase 2 assumes graph-resident, but #2516 is follow-on. If CI workflow assumes graph mappings exist before #2516 ships, every PR breaks on the first uncovered subdomain. Phase 2 contract requires fallback to filepath heuristic on graph miss. CI never depends on graph being complete; #2516 only improves accuracy.
Pre-commit narrows before format auto-fix is wired; agents commit unformatted code, Phase 1 verification PR fails on a different gate. Sequencing inside Phase 1 matters. Narrowing without auto-fix = formatting drift on main. Silas's Phase 1 step 4 explicitly switches format to warn-with-auto-fix in the same commit as the narrowing.
Deploy smoke + auto-rollback (#2333) drifts past Phase 2 close; deploy stage stays manual indefinitely. The brief's destination requires this; without it the to-be diagram is partially aspirational. Phase 2 exit explicitly requires smoke + rollback verified. Not "soon" — at Phase 2 close.
Either track silently rescopes mid-phase without informing the other. The plan only works if both tracks update it when scope changes. Otherwise this doc rots into fiction. Any phase-scope change goes through Jeff + a doc update. Re-review at each phase exit.
