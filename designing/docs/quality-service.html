Verdict — Bimodal, Not Inverted

The shape is wrong in the middle
7,394 unit tests (strong base). 193 E2E tests (decent top). 15 integration tests (collapse in the middle). The pyramid looks like an hourglass — heavy at both ends, hollow where service interactions happen. This is where "it works in my test" meets "it breaks in production."
Rust compensates for Chorus's missing layers
3,730 Rust unit tests + Rust's type system + 145 BDD scenarios effectively replace integration and E2E for the hooks layer. This works because the Rust core is a pure function pipeline — inputs in, decisions out. But the JS orchestration layer (Bridge, Clearing, board-client) has no equivalent safety net.
Coverage thresholds are enforced but misleading
Jest enforces 80/75/60 (lines/functions/branches). This catches coverage regressions but doesn't measure test quality. A handler with 90% coverage and zero integration tests is "passing" but fragile. The threshold prevents going backwards — it doesn't push forward.
The E2E layer is well-scoped
32 Playwright specs covering CRUD, auth, visibility, security, and content coherence. These test what Jeff sees. The gap is Chorus — Bridge, Clearing, and Werk have zero E2E coverage. Jeff uses these daily.
Performance is a blindspot
13 tests across everything. No SPARQL query time budgets (despite 23M triples in Fuseki). No page load regression. No API latency SLAs. When things get slow, Jeff notices before the system does.
