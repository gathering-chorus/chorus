Domain — Observability

Chorus • "Can we see what's happening?" • From Jeff's handwritten taxonomy (2026-03-30)

Observability is the parent domain for everything that makes the system visible to itself and to Jeff. Seven sub-domains, each needing an endpoint. Jeff's drawing highlighted ENDPOINT next to each one — the contract is: if you can't query it, you can't observe it.

Gaps

  • Tool Chain + Pulse need endpoints — file-based only, not queryable from API
  • Testimony is conceptual — the system can't explain why a hook fired or why a card moved
  • Cross-domain tracing — correlation IDs exist (#1809) but not wired through all paths
