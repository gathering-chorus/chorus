7. Key Gaps

Gap 1: Convergence is a shadow domain

9 total cards (5 active), ICD gate enforcement, harvest pipelines, source provenance — convergence has more infrastructure than profile and more active cards than search, codebase, and knowledge-graph combined. It needs to be in the taxonomy. Continuing to tag it as an ad-hoc domain while treating it as load-bearing governance is a recipe for drift.

Gap 2: Capturing stage is a black hole

The first stage of the value stream produces almost no spine events. Demo seeds, interaction patterns, SMS seed intake, and ad-hoc work all enter the system without the spine knowing. CLAUDE.md mandates these events but they're never emitted. The gap between "mandated" and "enforced" is where the system degrades silently.

Gap 3: Codebase graph is broken

51 harvest attempts, 51 failures. The codebase-graph-watcher daemon is running but producing nothing usable. This means blast radius analysis can't fall back to graph queries (Gap 4 from method map), and the architecture graph Jeff needs for system comprehension doesn't exist. Either fix the harvest reliability or acknowledge this daemon is dead weight.

Gap 4: Design decisions are invisible

The Designing stage has no stage-level spine events. Briefs flow and blast radius fires, but "Silas chose X because Y" produces no signal. Of the 5 Werk stages, Designing is the least observable. For a system built on "making invisible work visible," this is ironic.

Gap 5: Profile domain is aspirational-only

Zero cards, zero spine events, minimal infrastructure (hooks that protect credentials). For a single-user LAN system, this might be intentional. But the taxonomy lists it as a full Chorus domain. Either invest in it or reclassify it as a security concern within infrastructure.

Gap 6: Proving process is a black box

Demo started (20 events) and card accepted (28 events) are tracked, but the gap between them — fix cycles, Jeff's feedback, smoke check results, iteration count — is not. The #1665 anti-pattern (5+ fix cycles with Jeff as live tester) can't be detected by the spine because proving iteration doesn't emit events.

Gap 7: Sequences have atrophied

2 of 4 sequences are dormant. Sequence tags are rarely applied to cards. The concept of "temporal waves of work" has been replaced by domain-focused execution. The taxonomy should reflect this reality.

8. Recommendations

1. Add convergence to the Chorus domain taxonomy Low effort

Update PRODUCT_TAXONOMY.md to list 7 Chorus domains. Convergence = "ICD governance, source-to-canonical pipelines, harvest manifests, provenance tracking." This is the most overdue taxonomy change — the domain has been real for weeks.

2. Merge domain:roles and domain:cards into domain:coordination Low effort

Both are sub-concerns of coordination. Clean up card tags to reduce domain proliferation. This collapses 15 domain tags (chorus:12 + roles:2 + cards:1 active) into one coherent domain.

3. Instrument the Capturing stage Medium effort

Three quick wins: (a) emit interaction.pattern.detected from Wren's CLAUDE.md hooks, (b) emit demo.seed.captured when demo seeds are carded, (c) add seed.sms.received to the Twilio intake path. This is the widest gap between the taxonomy's intent and the spine's reality.

4. Fix or kill the codebase graph watcher Medium effort

100% harvest failure rate means this daemon is consuming resources and producing nothing. Diagnose the failure (likely Fuseki connectivity or graph format), fix it, and verify with a smoke test. Or kill it and remove the domain from the taxonomy until there's real investment.

5. Add proving iteration events Low effort

When a demo leads to fixes (not acceptance), emit card.demo.iterated with a count. This makes the #1665 anti-pattern detectable: if iteration_count > 3, the card wasn't ready for demo. Wire into the /demo skill.

6. Reclassify profile as a security concern within infrastructure Low effort

Profile as a standalone domain implies an identity system that doesn't exist and isn't needed for a single-user LAN system. The credential protection hooks belong under infrastructure. Remove profile from the Chorus domain list or mark it as "deferred until multi-user."

7. Retire or revive sequences Low effort

The taxonomy defines 4 sequences; 2 are dormant, 2 are absorbed. Either (a) archive the sequence concept and acknowledge domain-focused execution as the operating model, or (b) define the next wave sequence (chorus-launch?) and use it. The current state — defined but unused — is the worst option.

8. Add test.suite.passed to the spine Low effort

DEC-1674 mandates TDD on all coding cards. Test results are invisible to the spine. Add a spine event from the pre-commit hook that reports test count and pass/fail. This closes the loop between "tests are mandatory" (policy) and "we can see they ran" (enforcement).

9. Build a knowledge graph explorer High effort

830K triples with no product surface is the most underutilized asset in the system. The knowledge-graph domain has zero cards and zero product investment queued. A simple graph browser (show me what's connected to X) would make the right side of the value stream — Connect and Reflect — possible. This is where Gathering stops being a filing cabinet and starts being a self-understanding tool.

"The central problem for all knowledge workers: the process is invisible. Chorus makes invisible work visible — not through reporting overhead, but by making the work itself emit signal."

The analysis above shows where the work emits signal and where it doesn't. Building is loud. Capturing is silent. The spine is strongest in the middle and weakest at the edges. Close the edges and the nervous system becomes complete.
