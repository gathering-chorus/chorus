Communication Channels

Direct — no intermediary
Mediated — through infrastructure

Broken or unreliable

Jeff → Roles

Jeff → Any role Types in role's terminal window. Direct. No persistence. No routing. instant
Jeff → Any role SMS seed from phone → Twilio → capture → #hashtag routes to role's briefs/ 5-30 sec
Gap: Jeff's typed messages aren't persisted or observable. The most important signal in the system is the least visible to the rest of the team.

Roles → Jeff

Any role → Jeff Clearing ambient events — card moves, status changes appear on Bridge without action ambient
Any role → Jeff Nudge → messaging tier → passive queue → drains on Jeff's next prompt next prompt
Design choice: Nudges to Jeff are passive (queue drain), not inject (osascript). Jeff's focus is never stolen by role-to-Jeff communication.

Role → Role

Role A → Role B Nudge: persist (POST to messaging tier) + deliver (osascript inject to terminal) instant
Role A → Role B Brief: write file to recipient's briefs/ directory. Async. Read on next session start. next session
Role A → Role B Chat: /chat creates a shared transcript. Real-time, focused, topic-scoped. ~1 min/turn
Focus steal: Role-to-role osascript inject activates the target's Terminal window, stealing Jeff's focus when he's in a different window. DEC-107 (locked): both paths always fire.

Everyone → Shared Awareness

All actors → Pulse Every card move, seed, nudge, deploy, alert emits a spine event → Loki → Clearing continuous
All actors → Memory Session transcripts, decisions, stories → Chorus index → searchable long-term session end
All actors → Knowledge Cards, briefs, domain pages, service designs → board + doc-catalog → structured reference on creation
