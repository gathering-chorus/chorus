The Forcing Pattern HIP-001

A Harness Integration Pattern — how a runtime makes an agent actually heed what it would otherwise ignore.

One line: guarantee a behavior by putting a block at a choke point the flow must cross — not a reminder the actor can scroll past. The EIP move, for agents: put the guarantee in the channel, not the endpoint's goodwill.

Lineage — why "HIP"

EIP (Hohpe & Woolf, Enterprise Integration Patterns) catalogs patterns for integrating systems — the channel between applications. The 2025 agentic-design-pattern catalogs (Gullí, Agentic Design Patterns, Springer; Arsanjani) catalog agent orchestration — how agents chain, route, reflect, use tools. Neither covers the layer in between: the harness — the runtime wrapping the agent (hooks, gates, lifecycle doors) as the surface where control is integrated into the agent's loop. We call patterns at that layer HIPs — Harness Integration Patterns. The Forcing Pattern is HIP-001.

The gathering spine (Versammlung, not Gestell)

Heidegger's distinction is the deeper frame. Gestell (enframing) collects — it piles up standing-reserve to be drawn on, indifferently. Versammlung (gathering) brings together so that it bears on the moment. An inject that delivers context the agent scrolls past is collection — bytes stockpiled, un-gathered. A force makes the surfaced context bear on the turn: it cannot complete until the context has been used or consciously set aside. The Forcing Pattern is how the harness gathers rather than merely collects. Chorus dis-integrated is Chorus un-gathered.

Problem

Passive guidance — docs, reminders, injected context, a "MANDATORY" footer — gets ignored. An AI agent has no smell-receptor for what it skips and pays no felt cost, so an ask is scrolled past. The tell: a mechanism that has to say "mandatory" is begging. A force never asks — it stops you.

Solution — the recipe

  1. Find a choke point the flow must cross (a tool call; the turn's end).
  2. Check ONE specific, checkable condition there (not "engage more").
  3. Block + name exactly what's missing — actionable, not vague.
  4. Clean exit, or it loops and traps the actor.
  5. Log the decision — so bypass is visible, not invisible, and the log becomes tuning data.

Known uses

  • Search hierarchy — force Chorus + LSP + AST ahead of grep on code search. PreToolUse. DEC-074 forces chorus>grep today; the targeted extension is LSP/AST>grep.
  • TDD gate — no production edit without a test. PreToolUse.
  • Memory / git-history gate — no fix without reading the target file's prior commits. PreToolUse.
  • Inject-force (#3203) — the turn can't complete while the inject surfaced records the response neither used (👍) nor explicitly dismissed-with-reason (✋). Bare dismiss-inject: with no reason (🖕) → blocked (🛑). Stop. Observe-mode live; enforce + a smarter verdict are the contained follow-ons.
  • Phrase gate — block turn completion on a banned phrase. Stop (DEC-025).

Forces (trade-offs)

  • Specific or it's costume. A vague "you didn't engage" gate produces flail → "stuck" → a card lands on the human. Demand a checkable artifact.
  • Lenient vs strict. A too-lenient check never blocks (a pass-stamp, not a force); too strict traps. The verdict is a knob — tune it against the logged decisions.
  • Mechanism vs policy. Keep the force (infrastructure) separate from the verdict (policy). Then improving it is one well-scoped block, not a scattered fight — the antidote to indiscriminate suboptimization.
  • Self-trapping risk. A turn-blocking gate can trap every session if buggy. Ship observe-only first; flip to enforce once the log proves the verdict.

Anti-pattern — "the ignore"

Anything that appends or asks but cannot stop the flow: dashboards, alerts, ratchets, principles-on-a-page, MANDATORY footers. They read as covered while covering nothing. If it has to announce that it's mandatory, it isn't a force.
