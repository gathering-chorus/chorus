HIP-002 — The Generated Seam A Harness Integration Pattern · sibling to HIP-001 (The Forcing Pattern)

Wren · 2026-06-13 · the integration pattern behind athena-forge

One line: forge every surface from one model through one seam; weave cross-cutting control in at the seam, at generation time, and every surface — the agent's (AX) and the human's (JX) — inherits it by construction. The guarantee is in the generator, not in each endpoint's goodwill.

Lineage — why "HIP"

HIP = Harness Integration Pattern: a pattern at the layer between EIP (system integration) and the agent-orchestration catalogs — the harness, the runtime where control is woven into the agent's loop. HIP-001 (The Forcing Pattern) integrates control at a turn-door (a block at a choke point). HIP-002 integrates control at the generated seam — the single request point every forged route crosses. Where HIP-001 forces a behavior at runtime, HIP-002 bakes the concern in at generation time, so it cannot be omitted.

Problem

Hand-wired surfaces drift and leak. With N surfaces × M cross-cutting concerns (auth, authz, logging, validation, rate-limit), there are N×M places to get it right, and every one depends on an endpoint author remembering. Worse for agents: the agent's tool and the human's page are usually separate code paths over separate data fetches — so they disagree, and the disagreement is invisible until it bites. "Add auth to every route" and "keep the page in sync with the API" are asks; asks get skipped.

Solution — the recipe

  1. One model is the source (OWL/SHACL) — the single description every surface derives from.
  2. A generator forges every surface from it — routes (API), MCP tool, skill, page — so there is no hand-wired surface to drift.
  3. All generated routes pass ONE seam — the single request/response point (an IoC seam), not per-route plumbing.
  4. Inject cross-cutting concerns at the seam — auth, authz, logging, the HIP-001 force — born pre-wrapped; every route inherits them because every route crosses the seam.
  5. AX and JX render from the same generated API → they match on data by construction. One source, no second path, no drift.

Forces (trade-offs)

  • The seam must be the ONLY path. One hand-wired bypass route and the guarantee is gone — the whole value is that there is no other door.
  • Generation must be deterministic. Same model → same surfaces, or the seam's guarantees become non-reproducible.
  • Coherence flows from the model. A garbage model forges garbage surfaces uniformly — the seam amplifies whatever the model says, good or bad. (Hence the write-door + SHACL discipline; the seam assumes a coherent model.)
  • One seam, two postures. Localhost-default vs authenticated is a posture at the seam, not a per-route choice — flip it once.

Anti-pattern — "the hand-wired surface"

Per-endpoint cross-cutting (each route re-implements auth/logging) → N×M drift, and the one route someone forgets is the hole. Two data paths (the page fetches differently than the agent's tool) → guaranteed disagreement, invisible until a demo or an incident. If a surface isn't forged from the model through the seam, it is a place the system can quietly diverge from itself.
