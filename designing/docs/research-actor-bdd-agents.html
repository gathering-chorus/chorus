Actor-Driven BDD for Agent Coordination

Research position • Bridwell, Kade, Silas • April 1, 2026 • From practice, not theory

The Claim

A pipeline exists for specifying, testing, and enforcing AI agent coordination behavior that nobody has published: actor diagram → BDD scenarios → step definitions → runtime enforcement. Each stage is known. The connected pipeline is not. We built it today and it works.

The Pipeline

Actor Diagram domain page (HTML)
→
Gherkin BDD Jeff's experience
→
Step Definitions curl to live API
→
Implementation endpoint code
→
Runtime Gate Rust enforcement

Each stage produces an artifact that feeds the next. The actor diagram defines who does what. The Gherkin describes what the human experiences. The step definitions test the real system with real HTTP calls. The implementation follows the spec. The runtime gate enforces the contract in production.

What Makes This Different

The Feedback Loop Nobody Has

Jeff corrects a behavior → correction saved to memory → memory informs a gate → gate enforces the correction in every future session. The human's feedback becomes structural. Not a prompt tweak. Not a fine-tune. A gate that blocks the wrong behavior before it happens.

The BDD scenarios describe Jeff's experience, not internal system behavior. "When Jeff asks what happened on card #1930, then he gets a timeline" — not "when the API receives a GET request, then it returns JSON." The test is from the human's perspective. The agent can't argue with it.

Five Gaps in the Literature

1. Diagram → BDD for Agents

Nobody has published translating actor/sequence diagrams into Gherkin scenarios for AI agent behavior. Components exist separately.

2. BDD for Agent Behavior

BDD is used with agents (agents running Gherkin tests) but not for agents (Gherkin specifying agent behavior).

3. AUML for LLM Agents

Agent UML was purpose-built for multi-agent interaction diagrams (2000s). Nobody has revived it for LLM-era agents.

4. Human Feedback as Gates

Human corrections becoming structural enforcement (not prompts, not fine-tuning) is unexplored. 120 feedback memories → 33 hook modules.

Why It Works for Agents

"The actor-diagram → BDD → code pipeline would have completely broken the brains of most human teams I have led."
— Jeff Bridwell, after watching two cards ship in 2.5 hours

The discipline that breaks human teams — spending time on definition before code — is free for AI agents. No attention fatigue. No "can we just start coding?" The actor diagram took 5 minutes. The BDD took 3. The endpoint took 10. The definition work that humans resist because it's cognitively expensive costs agents nothing.

The human holds the frame ("do the actor diagram first"). The agents follow the path. The result: ideas survive the translation from concept to code because the definition phase doesn't consume the energy needed for implementation.

The Institutional Memory Angle

The pipeline produces institutional memory as a side effect. The actor diagram documents the domain. The Gherkin documents the expected behavior. The step definitions document the test contract. The gate enforces it going forward. When a new agent session starts, the entire specification is readable — not buried in prompts or configuration.

The system can now recall what the human has lived through, so the human doesn't have to re-teach it every session.

What Chorus Is

A coordination protocol for three AI roles and one human. 108 decisions. 1800+ cards. 33 hook modules enforcing behavioral contracts. Not a coding assistant — a delivery team. The pipeline described here emerged from practice: 5 months of daily sessions, 120 feedback corrections turned into structural enforcement, and one afternoon where the pattern crystallized.

The contribution isn't any single component. It's the connected pipeline — from a human's question ("what did Wren and I talk about?") through domain definition, behavioral specification, test-first implementation, and runtime enforcement — executed in one session by agents who don't get tired of doing the definition work that humans skip.
