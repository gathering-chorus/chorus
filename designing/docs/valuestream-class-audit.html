The verdict

The ValueStream class is a thin scalar shape with its relationships hung off generic, mis-typed, inverse-less predicates. That's why:

• nesting didn't work natively — the shape doesn't carry contains;
• I had to hand-populate both inStream and contains — they aren't owl:inverseOf, so they drift;
• the recursion (step→child stream) rides hasValueStream, which is undeclared (no domain/range);
• contains actually belongs to Domain, and stageOrder to Vertebra — both overloaded onto ValueStream.

The instances are fine; the class is wrong. Fixing it is a class-level model edit (declare contains on ValueStream→ValueStreamStep, declare hasChild/hasValueStream with domain+range, link inverses, fix the overloaded domains) — a shared-ontology shape change, so Silas-coordinated, on paper first.
