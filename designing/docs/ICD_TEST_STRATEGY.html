The Shift — From Human Governance to Automated Enforcement
At Staples: 30 people enforced the ICD
ARB reviewed ICDs quarterly. Canonical model team validated mappings. Developers hand-coded from spec. Integration testers caught drift. When humans didn't follow the process, the system drifted silently.
Staples Enterprise Integration Services, 2009–2016
At Gathering: Tests enforce the ICD
validateFromICD() is the ARB — automated, every write, no exceptions. icd-lint.py is the model team. generate-from-icd.py is the developer. Round-trip tests are the integration testers. Jeff makes tier decisions. Agents enforce them.
Chiron paper V4: "speed and quality improve together" when AC validation is automated
The Anzo Insight: Author where humans see, persist where machines query
Staples bought Anzo because business users could author mappings in Excel and the system persisted to RDF. Same pattern: SEMANTIC_MAPPER.html is the authoring surface, Fuseki is the persistence layer. Tests verify the round-trip between them — that's where drift kills you.
Cambridge Semantics / Anzo, Staples 2010–2011
Six Test Layers
1 Surface ↔ Graph Consistency Silas
The highest-priority layer. Three representations of the same truth must agree: OWL ontology in Fuseki, SEMANTIC_MAPPER.html rendering, TypeScript schemas in icd-validation.ts.
Tests: For each domain: SPARQL query field count = HTML parse field count = TS schema field count. Field names match. Severity tiers match. Source mappings match.
Trigger: Any change to SEMANTIC_MAPPER.html, icd-ontology.ttl, or icd-validation.ts.
Failure mode prevented: Someone edits HTML, forgets to update the graph. Or vice versa. The Anzo lesson — this is where enterprise integration programs die.
Existing: icd-lint.py (HTML only)
Gap: No cross-representation comparison. HTML ↔ RDF ↔ TypeScript drift is undetected.
Test: icd-consistency-test.py — SPARQL + HTML parse + TS AST → compare all three
2 Round-Trip Integration Kade + Silas
The full pipeline: ICD definition → code generation → validation gate → write record → read back → verify fields match contract. Yesterday's Social domain session (#1516), automated.
Tests: For each domain: generate-from-icd.py produces schema → validateFromICD() accepts a valid record → write to pod → read from Fuseki → all required fields present, all violation-tier fields non-null.
Trigger: CI on every commit touching src/validation/ or data/about/SEMANTIC_MAPPER.html.
Failure mode prevented: Generator produces schema that doesn't match ICD. Or validator accepts records the ICD says are invalid.
Existing: Manual — ran once for Social (#1516), once for Photos (#1515)
Gap: Not automated. Not in CI. Each domain onboarding is a manual walk-through.
Test: icd-roundtrip-test.ts — Jest integration test per domain
3 Severity Tier Enforcement Kade
Change a field from enrichment to violation → the gate starts rejecting records that previously passed. Prove the contract change propagates through generation → validation → write path.
Tests: Create a test record missing a specific field. Set field to enrichment → record passes. Change to violation → record rejected. Verify at each layer: ICD, schema, validator, write path.
Trigger: Any severity tier change in the ICD.
Failure mode prevented: Tier change in ICD doesn't propagate to runtime. The "soft middle" returns.
Existing: None
Gap: Tier changes are untested. We discovered thumbnailPath should be violation-tier — but nothing verified the change propagated.
Test: icd-tier-enforcement-test.ts — parameterized test per domain × tier transition
4 Cross-Domain Link Integrity Silas
A Decision references a Person who appears in a Story. All three resolve in the graph. Broken links surfaced as test failures, not silent nulls on a page.
Tests: SPARQL queries that traverse cross-domain links: Person → Stories mentioning them. Decision → Cards it spawned. Recommendation → Person who wrote it. Every link target must resolve to a real record.
Trigger: Any harvest or dedup operation that changes slugs or deletes records.
Failure mode prevented: Dedup merges "curt-hamm" into "curtis-hamm" but Stories still reference "curt-hamm" → broken link. (#1520 nearly caused this.)
Existing: None
Gap: Cross-domain references are unchecked. Dedup, harvest, and slug changes can silently break links.
Test: icd-link-integrity.sparql — SPARQL ASK queries for every cross-domain reference pattern
5 Convergence Proof Wren
For each domain: count(source records) → count(canonical records) → count(violations) → count(enrichment gaps). The numbers must be explainable. No silent data loss.
Tests: Source record count (files on disk or API response) matches canonical count (Fuseki) minus violations (rejected by gate) minus dedup merges. The equation must balance.
Trigger: After every harvest, dedup, or migration operation.
Failure mode prevented: 122,431 canonical photos but only 114,414 thumbnails. Where are the other 8,017? The test makes the gap visible and explainable.
Existing: validate-canonical.py (Photos only), manual SPARQL queries
Gap: Only Photos has a data quality report. No automated convergence proof for other domains.
Test: convergence-proof.py — per-domain source-vs-canonical reconciliation
6 Domain Onboarding Regression Kade + Wren
Add a domain to the ICD → generate code → validate existing data. If this breaks, the pipeline is broken. The full DEC-095 sequence as a test.
Tests: A synthetic "test domain" defined in the ICD with known fields and sample data. generate-from-icd.py produces a schema. validateFromICD() passes/fails the right records. If any step breaks, domain onboarding is broken.
Trigger: CI on every commit to generate-from-icd.py, icd-validation.ts, or harvest pipeline code.
Failure mode prevented: A refactor to the generator breaks domain onboarding. Nobody notices until the next real domain.
Existing: None
Gap: No synthetic test domain. Each onboarding is a first-time walk through unfamiliar territory.
Test: test-domain fixture in data/about/SEMANTIC_MAPPER.html + icd-onboarding-test.ts

Execution Model
Tests query Fuseki, not mock data
Production graph is the source of truth. "Production telemetry is the spec that survived." Mock data creates a parallel universe where tests pass but production drifts.
Charity Majors, "Production Is Where the Rigor Goes", March 2026
ICD-as-graph means the contract is testable as data
SPARQL can verify the ICD's own consistency. "How many violation-tier fields does Photos have?" is a query, not a parse. The ontology validates itself.
