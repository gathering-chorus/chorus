Key Principle: Don't Create Directories for Patterns

"Spine and Policy aren't codebases, they're behaviors implemented across multiple real domains. Creating directories for patterns creates empty directories." — Kade

The conceptual domains (Spine, Policy, Coordination) are real behaviors but not bounded contexts. They don't need directories, owners, or domain pages. They need the real domains (Gates, Hooks, Clearing, Cards) to implement them correctly.

Structural Moves

Move #1: Move flow.handler.ts and its data assembly from the Gathering app into chorus/directing/clearing/. The Clearing's data source shouldn't live in a different repo from the Clearing's UI. One domain, one repo, one owner.
Move #2: Rename board-client to board-service (or merge into platform/services/). It's a service with its own tests, CLI, and build — not a client library.
Move #3: Framework needs both a gate AND a workflow. Gate without workflow = friction. Workflow without gate = optional. Card #1918 (service design page) is the workflow. Extend ICD gate (#DEC-095) to all domain writes for enforcement.
