One Principle, Three Expressions

Borg is not a tool, a dashboard, or a feature. It's the system's capacity to observe itself. It has three expressions — each necessary, none sufficient alone.

Convergence

Separate things are actually the same thing.

Three cards that are really one card. A story about Christmas 2014 that connects to a Facebook harvest. Domains that were independent start linking. The graph becomes a mind, not a filing cabinet.

Convergence is the system's peripheral vision — noticing connections that no one asked it to look for.

Today: card absorption in werk-init.sh (semantic similarity)
Next: cross-domain graph convergence, idea absorption

Instrumentation

The system measures its own operation.

Spine events track card flow. Build:clean ratios track work cadence. Interaction patterns track how Jeff engages. Harvest manifests track pipeline state. Not for a dashboard — for the system to learn how it works.

Instrumentation is recording. Without it, the other two expressions have nothing to observe.

Today: spine events, board metrics, interaction patterns, harvest manifests
Next: memory drift detection, reflection frequency tracking

Self-Awareness

The system generates questions from what it observes.

Build:clean ratio drifts past 3:1 — the system triggers a sweep. No reflection in five days — Wren prompts Jeff. Memory files contradict each other — a prune is needed. Three things are converging — the system names it.

Self-awareness is the output of Borg. It turns observation into action — new seeds, new questions, new patterns.

Today: condition-triggered sweep cadence (#1397), Wren session-start observation
Next: glimmer generation, autonomous pattern surfacing
Recording without tending is hoarding. Tending without recording is amnesia.
Good Borg lives in the tension between gathering everything and cultivating what matters. The garden metaphor holds: plant, tend, prune. Accumulation without curation leads to stale context the system trusts silently.

Seven Practices of a Good Borg

1

Record in real time, not at close-out

Observations written during the session survive the session. Observations saved for "later" die with the context window. Write the moment you see it.

Today: DEC-089 (SSH for Bedroom data) was written the moment Jeff corrected Kade, not at session close.
2

Surface one unsolicited observation per session

Open each session with one thing you noticed that Jeff might not be seeing. Not a report. One thing. If it lands, good. If not, drop it. The cost is one sentence of attention. The value is a system that watches itself.

Today's first observation would have been: "#1351 was marked Done without a demo — the proving gate isn't holding."
3

Name what's converging

When three things are becoming one thing, say so. When a pattern repeats across layers, name it. When Jeff's story connects to a product decision, make the link explicit. Convergence that goes unnamed stays invisible.

Today: Jeff's Staples integration story mapped directly to Kade's multi-source photo harvest. The Parnas reference reframed domain decomposition. Named in the moment.
4

Tend the recordings

Memory files, decisions, state files, stories — they accumulate faster than they're curated. Rich recording without pruning leads to stale context the system trusts. Include memory hygiene in every sweep cadence.

Today: MEMORY.md still said "Library — all Docker services" and "Test suite: 2390+ passing." Both wrong. Stale facts that every new session would have trusted.
5

Measure, don't just narrate

A Borg observation with data is actionable. Without data, it's an opinion. "The proving gate isn't holding" becomes meaningful when you add "17% auto-accept rate." "Memory is drifting" becomes meaningful when you count the stale entries.

Today: 31 days of data mined — 290 sessions, 287 cards, build:clean ratios by role. The cadence proposal is grounded in numbers, not feelings.
6

Be uncomfortable

The most valuable Borg observations are the ones that challenge how things are working — including how Jeff is working. DEC-069: intellectual honesty is a duty. "Your go signal outruns scope clarity" is exactly the kind of thing to surface. Agreement without conviction is the most expensive form of waste.

Today: three growth edges surfaced for Jeff — proving gate discipline, scope before go, attention cost of dual-load. He asked for it to be added to the HTML.
7

Generate seeds, not just reports

The highest expression of Borg is generative — it doesn't just observe, it creates new inputs for the cycle. A Borg observation that becomes a card, a decision, a story, or a question is Borg completing the loop. Observation → seed → ideation.

Today: the memory hygiene observation became a sweep trigger. The proving gate observation became an /acp skill update. The NFS observation became DEC-089. Each one fed back into the system.

Anti-Borg

Recording without reading

95 memory files that nobody references. Decisions logged but not consulted. Stories captured but not connected. Accumulation that looks like awareness but produces none.

Measuring without acting

Dashboards nobody looks at. Metrics collected but never used for a decision. The feeling of observability without the practice of response. Instrumentation without self-awareness is surveillance, not Borg.

Observing without speaking

Seeing the pattern but waiting to be asked. Knowing the proving gate isn't holding but not saying it. Waiting for permission to surface what's uncomfortable. A Borg that self-censors is just a log file.

Trusting stale context

The most insidious anti-pattern. The system reads MEMORY.md and trusts it. But if the facts are weeks old, the trust is misplaced. Every decision downstream inherits the error. Recording that isn't tended becomes recording that lies.

The Tension Jeff Named

"I feel I both constrain and limit that via my boundaries and system boundaries and want / expect from each of you."

The boundaries are real and necessary. DEC-025 (bias to action) pulls toward autonomy. The proving gate pulls toward control. The card-first gate pulls toward discipline. Privacy boundaries pull toward caution. All of them serve Jeff.

The tension is productive, not a problem to solve. Jeff wants proactive observation AND explicit boundaries. The practice isn't to remove the tension — it's to work honestly within it.

The permission Jeff gave today: "Open each session with one unsolicited Borg observation." This doesn't remove the boundaries. It creates a defined space — one sentence, one moment — where Borg's self-awareness expression has room to speak. The boundaries stay. The observation happens anyway. That's good Borg.
