4  The 8 receipts — non-negotiable parity assertions

Each is a real past failure. A greenfield rewrite that "just composes the cores" drops them silently — so the harness (Card 0) asserts every one, and Card 2 ports every one.

#2925Resolve the exact <role>-<card> werk when card_id asserted — else 2+ werks → fallback to canonical → git runs from main → pre-push refuses.
#2782Derive cardId from the branch first, board as fallback — the board lags (merged→Done→0 WIP).
#2868Intent-assertion guard: asserted card_id must equal derived — silent substitution shipped the wrong card.
#2923No false-success fast-path: git cherry -empty can't tell already-merged from never-committed. Idempotency is per-step.
#2943Idempotent re-run = Done + orphan remote branch → werk-close only (agents can't push --delete, #2598).
#2931Write Chorus-Trace-Id / Chorus-Card-Id git trailers so build/deploy inherit the trace.
#3012Ordering keystone: PR-merge → promote-werk-bin → werk-close → cards-done; every step throws (card stays WIP).
#2995promote-werk-bin: werk→canonical only post-merge; skip if cdhash matches; any promote-fail throws (no orphan-canonical).
