Shared Awareness

2026-04-04 — Silas (#2022), built with Jeff. Source: Jeff's graph-paper sketch.

Shared Awareness is the system's ability to know its own state. When it breaks, Jeff becomes the monitoring system. Five components from Jeff's sketch: Time, Pulse, Memory, Knowledge, System Context.

Purpose

L2 in the Protocol Stack. Not a domain — a capability that spans Observability, Spine, Search, and Memory domains. Provides the guarantee: any role can answer "who is active, what are they doing, and when did they last emit?" without asking Jeff.

Forcing Functions

Components don't help if roles forget to use them. These are mechanical — no opt-in required.

Compound loop: fires on every UserPromptSubmit. The role doesn't choose to search — the hook does it for them. Zero discretion.
Ops awareness: fires on every PostToolUse (Bash/Read). If the API is down or alerts fired, the role sees it. 60-second cooldown prevents spam. Exit code 2 ensures visibility.
Alert rules: run every 60 seconds via LaunchAgent. Time-gated (don't fire before deadline), cooldown-gated (don't spam). Fire to Bridge so Jeff sees it without checking.
Skills in git: all 31 skills symlinked from platform/skills/. Edits are committable, diffable, shared. No invisible drift between sessions.

Open Questions

  • Session watcher latency (#2021): fswatch misses new session creation for up to 4 minutes. Direct JSONL tailing in gemba-tick works around it, but the indexer still lags for Chorus search.
  • Heartbeat/liveness: No mechanism to distinguish "idle" from "session crashed." Role state file says "waiting" but doesn't confirm the session is alive.
  • Desk freedom test: L2 passes when roles can detect each other's presence without Jeff. Not there yet — roles still can't reliably answer "is Kade's session alive?"
