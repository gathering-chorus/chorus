Memory Retrieval Orchestration

Retrieval is an active process, not passive recall. Orchestrated, not accidental.Active

⚡
Trigger Layer
What activates retrieval
Decision point (card creation, priority call, architecture choice) Pattern match (current situation resembles a past one) Role context (session start loads role identity + preferences) Jeff signal (story shared, question asked, direction given) Time-based (session start reconciliation, daily standup)
🔍
Search Layer
Where retrieval looks (DEC-074 order)
Chorus index (96K messages — semantic search across all sessions) Codebase graph (Fuseki SPARQL — structural relationships) Memory files (stories, feedback, user, project memories) State files (decisions.md, backlog.md, projects.md) Git history (who changed what, when, why)
🧠
Synthesis Layer
How retrieved memories inform action
Shape matching: “this feels like X” (Staples → Docker migration) Value alignment: “Jeff would want Y” (independence → DEC-025) Pattern recognition: “last time we did Z, it went Q” (cruft velocity) Relational context: “this connects to person P” (Aubrey → recovery → Ginsburg)
⬆
Delivery
How memories reach the surface
Implicit: role thinks differently because context was loaded Explicit: “this connects to your Staples experience” Prompted: Jeff asks “do you remember...” and retrieval fires Ambient: morning practice check, session start state

The Gaps

Nodes exist but not edges. Capture depends on conversation. Durability is fragile.

💨
Missing Edges — Personal Memory
Connections between memories aren't captured. The graph of experience is disconnected nodes.
Ginsburg Aubrey Recovery Julian Parenting values
These edges exist in Jeff's mind. They don't exist in the system.
🔌
No Durable Event Stream — Team Memory
chorus.log was being rotated and truncated. The append-only spine is not append-only. Search was fragile — chorus.db hit 0 bytes. Git history is durable but not queryable as an event stream.
🎭
Sensory/Embodied Has No Home
The weight of a dead dog. Sun angle in winter. How walking changes thinking. These are real memories with real product implications (Self ontology, wellbeing signals, biometrics) and no capture mechanism at all.
🎙
Capture Depends on Conversation
If Jeff doesn't say it during a session, it's gone. SMS seeds help, but they're about ideas, not memories. There's no prompt, no ritual, no ambient capture for the experiences that shape product decisions.

Gaps — Not Yet Carded 8
Sensory/Embodied Memory Capture
No mechanism to capture physical experience — walking, weight, sun exposure. Transposition rate is a start but only covers typing stress.
Retrieval Orchestration
Active retrieval at decision points. When a decision is being made, proactively surface relevant memories. Currently ad-hoc — roles search when they think to.
Connection Discovery Automation
Edge persistence writes edges when discovered, but discovery is still manual. Could use embedding similarity to suggest connections.
Ambient Memory Prompts
Morning practice check is the first example. Expand: session start surfaces relevant stories, time-based prompts for follow-up.
Cross-Domain Memory Linking
Photos to stories, music to episodes, documents to decisions. The ontology supports it (#1401) but no harvester writes cross-domain edges yet.
Memory Durability Guarantee
chorus.log rotation is fixed but no formal durability contract. What gets archived vs rotated? No nightly backup of memory files.
PictureThis Plant Inventory Harvest
Jeff uses PictureThis for plant ID. Need to check API/export and build a harvester for the garden domain.
Story-to-Story Connection UI
Stories are in the graph but no UI to browse connections. “Show me everything connected to Ginsburg” should be a page.
