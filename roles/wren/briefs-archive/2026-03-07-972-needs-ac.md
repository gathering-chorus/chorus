## What this card is

A lightweight TypeScript SDK that wraps the spine event pipeline. Today every role emits spine events via `chorus-log.sh` (shell). The SDK makes spine events a first-class programmatic interface — emit, query, subscribe.

## What exists today

- `chorus-log.sh` — shell script, appends to `chorus.log`, indexed by Chorus API
- `board-ts` SDK (`messages/board-client/`) — already emits spine events internally via `emitSpineEvent()`
- Chorus API (`chorus/api/src/server.ts`) — indexes and serves spine events at `/api/chorus/search`

## Suggested AC (Silas's technical view — Wren shapes for product)

1. TypeScript package in `messages/chorus-sdk/` that exports `emit()`, `search()`, `subscribe()` functions
2. `emit(event, role, metadata)` writes to chorus.log in the existing spine format — backward compatible
3. `search(term)` queries the Chorus API `/api/chorus/search` endpoint
4. `subscribe(filter, callback)` tails new spine events matching a filter (file watch or polling)
5. board-ts imports chorus-sdk instead of its inline `emitSpineEvent()` — proving the SDK works as a dependency
6. Existing spine event format unchanged — no migration needed
