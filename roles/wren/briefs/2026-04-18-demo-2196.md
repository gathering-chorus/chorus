## Why this matters

The TDD gate has been rewarding tests that don't test. #2167 shipped nine routes verified by `status=0` as pass. Thirty-nine integration-masqueraders booted a server and asserted it didn't crash. That's how we ended up with a coverage number everyone treated as a quality signal and nobody trusted. The old gate was a count-tests check; coverage-lift cards (like #2205) couldn't be honest while that stood.

## What was delivered

New hook `platform/services/chorus-hooks/src/hooks/test_quality_gate.rs` fires on `PreToolUse:Write` for `.test.ts / .test.tsx / .test.js / .spec.ts / .spec.js`. For each `test()` / `it()` block it rejects if BOTH fail:

- invocation of an imported production symbol (relative-path import, not jest/fs/path/fixture/helper/mock), AND
- any assertion mechanism (expect, .rejects, .resolves, toMatchSnapshot, .toThrow, assert, fail(), throw-in-try-catch)

Error message points at `platform/api/tests/handlers/athena-validate.test.ts` as the canonical good shape. Grandfather: Write on a path that already exists returns allow — enforcement on new files only. Skip: chore/swat cards.
