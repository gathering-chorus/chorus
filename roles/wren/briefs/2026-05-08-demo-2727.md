**Date:** 2026-05-08 Boston

## What changed for the user

Before: nudges flowed sender → chorus.log → 2s-cadence LaunchAgent poll → chorus-inject keystroke. State of "is this nudge delivered" was derived from spine event fold; messages.db accumulated 10,500+ pending rows because the old delivery path was retired and nothing took over the drain.

After: nudges flow sender → POST /api/nudge → pulse stores row → async delivery worker calls chorus-inject → markDelivered or markFailed with typed error. Every nudge gets a UUIDv7 trace_id at sender mint, propagated via X-Chorus-Trace-Id header, stored as indexed messages.db column, stamped on every spine event. One Loki query returns one nudge's full lifecycle.

For Jeff in normal use: nothing visible — keystrokes still land the same way. For debugging: the "where is this nudge in the lifecycle" question becomes one query instead of a fold.

### Trace_id correlation (after #2765 merge + canonical rebuild)

A nudge sent silas → wren with trace_id `018f-xxxx-xxxx`:

```sql
SELECT trace_id, "from", "to", content, delivery_status, delivered_at FROM messages WHERE trace_id = '018f-xxxx-xxxx';
```

```logql
{event=~"nudge\\..*"} | json | trace_id="018f-xxxx-xxxx"
```

Both return the same nudge — DB has the row + terminal state, Loki has the chronological event stream. Spine ↔ DB join is structural via the indexed trace_id column.

## Architectural deltas captured

- **Mitigation A dual-emit**: sender emits BOTH `nudge.requested` and `nudge.emitted` during the read-side migration window. Closes the silent-observability-gap (Silas ops review during #2763). Cleanup card retires `nudge.emitted` after readers cut over.
- **Opt-in scanAndRequeue at boot**: `PULSE_REQUEUE_ON_BOOT=1` env var. Default unset = no stampede on deploy. Boot fail-fast on smoke OR scan errors. Three startup events emitted: `requeue.skipped`, `requeue.start{pending}`, `requeue.complete{enqueued}`.
- **PERMANENT_REASONS narrowing**: only `no claude window found` is structurally classified by chorus-inject (per Kade gemba). tcc-denied / window-ambiguous / encoding-error treated as transient — retry-until-exhausted is more honest than false-permanent.
- **selfTest via --count-windows**: chorus-inject doesn't have `--self-test` flag. Worker boot smoke uses `--count-windows claude` which exists and exercises the binary + TCC path.
- **chorus_log field validation**: nudge.* events refused at call site if missing required fields. Substring match (caller-controlled payload order; bounded false-positive); upgrade to comma-split parser if observed in production. Rust-side only; TS worker emits via structured JSON.
- **emitSpine timestamp field**: pulse worker writes `timestamp:` (canonical chorus.log convention) — earlier wrote `ts:`; one-line fix during #2764 cutover verification.
