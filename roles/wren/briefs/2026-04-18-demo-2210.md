## Why this matters

#2196 shipped with two doors open that Silas named at review:

1. **Partial-write race** — a Write of an empty `.test.ts` stub passed (no blocks to check), then a subsequent Edit could add bad tests because Edit wasn't gated.
2. **Parser fail-open** — unbalanced braces or mid-edit syntax returned zero blocks from the parser, and the Write passed silently even when `test(` / `it(` text was clearly present.

Both close with f08e321e. The gate now reads on-disk content, diffs block signatures (name + whitespace-normalized body) against incoming, and only quality-checks blocks that aren't grandfathered. Parser fail-closed refuses Writes where keyword-count exceeds parsed-block-count, with a pointer to `npx tsc --noEmit`.

## What was delivered

`platform/services/chorus-hooks/src/hooks/test_quality_gate.rs`:

- `analyse_incoming(existing, incoming)` replaces single-source analyse. `analyse()` kept as a thin wrapper so #2196 tests continue to work unchanged.
- `count_test_keyword_occurrences()` scans for `test(` / `it(` / `.skip(` / `.only(` at word boundaries. If count > parsed blocks, fail closed.
- `block_signature()` = name + whitespace-normalized body. Grandfather identity survives reformatting but not content change.
- `resolve_test_file_change()` handles both Write (existing = disk content) and Edit (incoming reconstructed by applying old_string → new_string to disk content).

+10 unit tests (31 total): diff grandfather/reject/allow cases, partial-write-race fixture (exact Silas scenario), whitespace-reformat grandfather, parser fail-closed cases, keyword-boundary respect, signature-sensitivity.
