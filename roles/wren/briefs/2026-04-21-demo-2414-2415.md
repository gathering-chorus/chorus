## #2414 — Boot contract drift e2e (zone a)

**File:** `platform/tests/boot-contract-drift-e2e.bats` — 10/10 green

**Why this matters:** the Rust unit tests (`session_init_gate_binary.rs`, `session_init_gate_recovery.rs`) prove the gate logic in-process. Before this card, nothing proved the *deployed* `chorus-hook-shim` binary on disk honored that logic. A drift could be introduced by a bad cargo build or a stale binary, and every role would boot green without anyone noticing.

**Safety:** every test uses a `snapshot_marker_state`/`restore_marker_state` setup+teardown pattern mirrored on `MarkerGuard` from the Rust tests — a test that corrupts `/tmp/claude-session-init/silas.{pending,done}` cannot strand my live session even if it panics.

**What to show:** `bats platform/tests/boot-contract-drift-e2e.bats` runs 10 tests, all green, exercising the real binary via `CHORUS_INJECT_DRY_RUN` style hermetic seams. Source-shape assertions also verify the three violation types (MissingStamp / VersionMismatch / Stale) and the sha256+sorted canonicalization pin.

## #2415 — Nudge persist+deliver DEC-107 e2e (zone b)

**File:** `platform/tests/nudge-dec107-e2e.bats` — 8/8 green

**Why this matters:** DEC-107 locked "persist AND deliver, both paths, every nudge" after 4 revisits. A source-gate Rust test (`nudge_force_source_gate.rs`) prevents the shape from regressing, but nothing behaviorally proved that *one* `nudge` invocation produces *both* a messages.db row AND a `role.nudge.delivered` spine event. This is the invariant Jeff cares about — "did the role actually get nudged" — not code structure.

**Shared-trace invariant:** the most interesting test (8) asserts that `role.nudge.sent` and `role.nudge.delivered` share exactly ONE trace_id for a given marker. Two distinct trace_ids would mean the paths fired from different invocations (a cycling pattern DEC-107 explicitly forbids).

**What to show:** `bats platform/tests/nudge-dec107-e2e.bats` runs 8 tests, all green. Hermetic via `CHORUS_INJECT_DRY_RUN=1` — no osascript side-effects.
