# Demo Brief: #2575 — Quiet team-coordination output

**Date:** 2026-04-29
**Builder:** Wren
**Card shape:** platform/tooling (code + quality + arch; no product/ops — no user-surface or service change)

## Why it matters

Jeff's window saturates when intra-team coordination renders the full body of every nudge and chat message verbatim. The body lives in the command-line argument and the harness UI shows commands verbatim — perceptual load even when not consciously reading. Named explicitly today: "5–7 minutes of nudging between the 3 of you and all the words were showing in claude — a lot of motion that I really can't even read or track."

## What shipped

- `platform/scripts/nudge` accepts `--body-file <path>` and `--body-stdin` flags. Backward compatible — existing positional usage unchanged.
- `platform/scripts/chat.sh say` accepts the same flags with the same semantics. Backward compatible.
- Body is read from file or stdin and substituted into the underlying call to `chorus-hook-shim`. Recipient delivery + messaging-API persistence unchanged. Only originator's visible command line is shortened.
- Memory file documenting the new pattern: `feedback_use_body_file_for_long_messages.md`. MEMORY.md index updated.

## AC verdict

5/5 met:
- AC1 nudge --body-file / --body-stdin — verified by three round-trip test calls; identical one-line output across all three forms
- AC2 chat.sh say --body-file / --body-stdin — verified by three test calls into a self-chat; transcript shows all three messages persisted correctly
