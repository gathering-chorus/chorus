## Why this matters

TEST.md (#2213) stated the binary rule in prose: every test file is hermetic-always OR integration-gated-at-file-level — no env-var conditionals inside a single file. Without enforcement the rule was aspirational; the 132 skipped describes across 5 mixed files exist because nothing stopped them at write-time. With 09454c73 live, any *new* `.test.ts` that branches describe/test/it on `process.env` gets rejected at Write. Legacy violators stay grandfathered (#2214 cleanup handles those opportunistically); new ones can't be introduced.

## What was delivered

`platform/services/chorus-hooks/src/hooks/test_quality_gate.rs`:

- `check_binary_rule(source)` detects two shapes:
  - Ternary routing: `process.env.X ? describe : describe.skip` (and `?=` / `===` equality variants, and it/test mirrors)
  - If-branch wrapping describe/test/it: `if (process.env.X) { describe(...) }` including negated
- Runs **before** the block-level check so structural violations get the right fix message ("split the file"), not misdirection to "your test has no assertions."
- Grandfathered: if existing content already has the violation, unrelated edits pass through (forces sweep-via-#2214, not blocking on legacy files).
- Allowed passthroughs: simple assignment, default-value short-circuit, env reads in test bodies, unconditional `describe.skip` at file level.
- Extension parity with #2196: same `is_test_file_path` helper gates all 5 extensions.

Also folded in the #2210 review item from Silas (2026-04-18 18:32, point 1): named drift-check fixture `diff_rejects_same_name_hollowed_body` locks the signature = name + body invariant.
