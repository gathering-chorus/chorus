# Demo: #2261 — Clearing: surface SWAT lane

**Builder:** Wren
**Date:** 2026-04-19

## Why it matters

SWAT cards were invisible in The Clearing — Jeff could see a role's WIP card in the tile but not their active swat. Today #2257 and #2261 both showed as UNCATEGORIZED in the board view but nowhere in the Clearing. If Wren is working a swat, Jeff couldn't tell from the tile.

## What shipped

**pulse.rs** — `parse_section_list()` added. `assemble_board()` now fetches both WIP and SWAT sections from `cards list` stdout in one pass. Board JSON now includes `swat_cards: [{id, title, owner, domain, status: "SWAT"}]`.

**tiles.ts** — reads `swat_cards` alongside `wip_cards`. SWAT cards appended to tile's cards list with `[swat]` suffix so they're visually distinct. WIP primary card unchanged.

## Smoke

```
pulse shim run → board.swat_cards: [{id: 2263, title: "[swat] gate:product demo check...", owner: "Wren", status: "SWAT"}]
Clearing tiles: Wren tile shows #2261 (WIP) + #2263[swat]
WIP cards: 3 — no regression
```

## Jeff sees

Wren tile in The Clearing now shows both active WIP card and any SWAT cards alongside it, tagged `[swat]` to distinguish from WIP.
