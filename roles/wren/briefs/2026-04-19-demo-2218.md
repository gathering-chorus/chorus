# Demo brief — #2218 Codesign with stable identifiers

## Why this matters

Every cargo rebuild of chorus-hooks emitted a new hash-based binary identifier (e.g., `chorus_hook_shim-d1da85efc2bf3103`). macOS TCC treats each new identifier as a new binary and can re-evaluate Accessibility grants, occasionally revoking them and stopping nudge delivery without warning. Stable identifiers prevent this class of failure.

## What shipped

- `platform/scripts/build-signed.sh` (pre-existing) — canonical build command that runs cargo + codesign with stable identifier
- `platform/services/chorus-hooks/Makefile` — updated `make build` to delegate to `build-signed.sh` (no duplicate logic)
- `platform/services/chorus-hooks/BUILD.md` + `platform/services/chorus-inject/BUILD.md` — document canonical build invocation, "don't use cargo build directly" warning, and honest scope boundary

## Scope boundary (honest)

This does NOT fix overnight TCC re-validation (covered separately in RCA #114). This only prevents the rebuild-triggered class of identity churn. AC and card description both state this explicitly.
