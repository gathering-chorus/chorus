# Demo Brief: #2160 — TDD + Demo Gates Not Firing

**Date:** 2026-04-19  
**Builder:** Kade  
**Status:** Done  

## Why It Matters

The gates enforcing TDD and demo discipline were dark since April 17. Any role could write production code before a test, or mark a card Done without a demo, and the shim returned `{"exit_code":0}` — silently allowing. Every quality claim we made during that window was unverified.

## Root Cause (Three Compounding Issues)

1. **done-gate.sh** added `exit 0` for nonexistent cards (commit c419e708). The original test used card 9999 which doesn't exist → gate allowed.
2. **tdd_gate** doesn't fire on `cards done` — it fires on Write/Edit of production code. The test label was wrong; both assertions tested demo_gate behavior.
3. **tdd_gate** requires `session_id` in the hook payload. Without it, `has_test_file_edit()` returns `true` (assumed pass) → gate allows.

**Bonus find:** done-gate.sh Evidence 2 (Chorus search) was producing false positives — the search telemetry for the query contains the search terms, so any subsequent search for the same card ID finds the prior search log. Fixed by adding `DONE_GATE_SKIP_SEARCH=1` test mode and tightening the grep to match structured JSON (`"event":"card.demo.started"` not free-text).
