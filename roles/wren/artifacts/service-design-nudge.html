Service Design — Nudge Delivery

Card #1793 • Owner: Silas • Language: Rust (chorus-hook-shim) • Domain: coordination, messages

1. Experience

I nudge a role and it arrives instantly in their terminal. I never wonder if it landed.

3. Integration Path

ONE path. No fallback. No passive drain. No ambient polling.

Sender rolecalls chorus-hook-shim nudge
→
Rust binaryparse args, resolve target
→
Persist to messaging tierPOST /api/nudge (history only, fire-and-forget)
→
osascript injectmatch window name → keystroke
→
Target terminalmessage appears in session

Key constraint: Persist is for history/auditability/memory. Delivery = osascript inject ONLY. Persist fires before inject (fire-and-forget, failure doesn't block). If inject fails, the nudge is NOT delivered. Persist succeeding does not mean delivered. On inject failure, message also written to /tmp/voice-inbox/<role>/pending-inject.txt as receipt.

Window matching: Each role has a Terminal window containing both the role directory name AND "claude" in the window title. Match is: window name contains product-manager (Wren), architect (Silas), or engineer (Kade). Jeff's windows do NOT match — nudges to Jeff go to passive queue only, no inject.

4. Non-Functionals

IdempotencyNot idempotent. Nudges are imperative — same message sent twice = delivered twice. No dedup.
SLA2 seconds from send to visible in recipient terminal.
ObservabilityEvery nudge emits spine event: nudge.sent with to=<role>,mode=injected|bridge|inject-failed
Error VisibilityFAILED with reason on stderr. Never silent. Never "ok" when it didn't work.
MonitoringHealth check LaunchAgent every 5 minutes. Verifies: (a) each role window exists, (b) canary inject actually arrives. Posts red to Bridge on failure.
Testabilitynudge --dry-run <role> "message" shows what would happen without injecting.

5. Failure Modes

Jeff must NEVER see: Silent drop — nudge says ok/DELIVERED but nothing arrives. This was the bug for 4+ rewrites. Zero tolerance.
Jeff must NEVER see: Wrong terminal — nudge lands in Jeff's window or a different role's window. Window matching must be unambiguous.
Jeff must NEVER see: Delayed delivery — arrives on next prompt instead of now. That's the passive drain bug. Eliminated by removing drain-as-delivery.
System handles: Role terminal not open — FAILED with "no claude window found for <role>". Sender sees it, sender decides what to do.
System handles: Role session not running — same as above. Window exists but no active session. Inject fires, Claude processes it on next cycle.

6. Test Contract

Derived from the experience statement. If the test doesn't match the experience, the test is wrong.

  • Unit: nudge to wren → DELIVERED + message appears in wren terminal
  • Unit: nudge to kade → DELIVERED + message appears in kade terminal
  • Unit: nudge to silas → DELIVERED + message appears in silas terminal
  • Unit: nudge to nonexistent role → FAILED with reason
  • Unit: nudge to jeff → QUEUED (passive), NOT injected into any terminal
  • Unit: --dry-run shows target window without injecting
  • Integration: Health check verifies each role has exactly 1 matching window with correct target pattern
  • Integration: Health check cross-checks window name doesn't contain another role's pattern (wrong-target detection)
  • Integration: Persist to messaging tier — nudge content queryable via curl localhost:3475/api/nudge/<role>/pending
  • Regression: Old paths deleted — no lsof, no find_role_pid, no get_tty, no TTY detection in delivery chain
  • Regression: All logic in Rust — bash wrapper is 2-line exec only (DEC-100)
  • Edge (Kade): Nudge during active tool call — does inject queue or interrupt?
  • Edge (Kade): Nudge to role that's rebooting — between session end and new session start
  • Edge (Kade): Concurrent nudges from 2 roles to same target — race condition on osascript inject
  • Edge (Kade): Nudge with special characters — URLs, quotes, backticks (Apple Music link pattern)

7. Dependencies

UpstreamAny role's session calling the Rust nudge binary (direct or via hooks shim)
DownstreamTarget role's Terminal window (osascript access required)
HistoryMessaging Tier (localhost:3475) — persist for audit trail, NOT for delivery
MonitoringBridge (localhost:3470) — health check posts alerts here

Critical dependency: macOS Terminal.app with Accessibility permissions for osascript. If Terminal is not running or permissions are revoked, ALL nudge delivery fails loud.

8. Design Gate

  • Experience statement written
  • Integration path mapped with spine events
  • Key design constraints documented (one path, no fallback, window matching rules)
  • Non-functionals specified
  • Failure modes enumerated (3 never-see, 2 system-handles)
  • Test contract verified (core + integration verified, edge cases carded)
  • Dependencies documented
  • Old paths deleted from codebase

Governing Decisions

DEC-107Nudge dual-path — SUPERSEDED by this design. One path (osascript inject). Persist for history only. The dual-path design caused silent failures for 4+ rewrites.
DEC-100No bash APIs — SATISFIED. All logic in Rust binary. Bash wrapper is 2-line exec only.
