7. Key Gaps

1. Integration class missing
Jeff's Staples ICD pattern (US9552400B2) isn't formalized. Integration has: sourceService, targetService, fieldMappings, schemaHash, validationRules. It bridges Services the way domains bridge ValueStreams. The icd-instance-*.ttl files exist in Fuseki but the ontology doesn't model the concept. — Kade
2. Card/Brief/Experience individuals = 0
Card class defined, 1800 cards on the board, zero in the ontology. Same for briefs and experience statements. The schema exists but the data isn't harvested. This is the largest volume gap.
3. NonFunctionals class empty
Every service should have idempotency, observability, error visibility, monitoring attributes. The class exists, zero individuals wired. This is where the real ops gaps hide.
4. hasGate is a string, should be ObjectProperty
Can't query "which practices are ungated?" because the gate link is unstructured text. Needs to be a relationship to a Gate individual with its own enforcement level (block/warn/none). — Kade
5. 5 services have no disposition
Nudge Delivery, Chorus API, Chorus Hooks, Messaging Tier, Session Tailer. Status × Disposition matrix has holes. Every service needs a conscious stance: abandon, tolerate, refactor, or rewrite.
6. Ontology not in Fuseki
The OWL exists as a file. It's not queryable by SPARQL, not accessible to hooks, not connected to the live system. Loading into Fuseki is the step that converts it from artifact to infrastructure.
