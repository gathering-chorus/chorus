OAGIS Business Object Documents (BODs)

The canonical model follows the OAGIS Verb+Noun pattern. Each BOD has an ApplicationArea (routing metadata) and a DataArea (business content).

VerbNounsDirectionDescription
SyncSalesOrder, Invoice, Shipment, ItemMaster, PartyBidirectionalSynchronize master/transactional data between systems
GetAddressVerification, Credit, Quote, ProductAvailabilityRequestQuery for information from a system
ShowAddressVerification, Credit, Quote, Shipment, ProductAvailabilityResponseReturn requested information
UpdateCreditCommandModify existing records
CreateCNHServiceRequestCommandCreate new records (Consumer & Non-Human health)
ConfirmCNHServiceRequestResponseAcknowledge creation
LoadPayableCommandLoad financial records into target system
Prior art: This is the same canonical model pattern Jeff Bridwell implemented at Staples (2010-2015) — ICD contracts between systems, canonical model as the hub, source-to-canonical maps enforced before any code is written. The OAGIS standard provides the noun vocabulary; the ICD provides the field-level contracts.
