# Chorus extraction — prep-then-move sequence

**#3097 deliverable (2026-06-13).** The durable record of how chorus code leaves the Gathering repo without a half-migrated state. Authoritative sequence: ADR-041 §6; family overview: card #2291.

## The rule

Extraction is **regenerate-or-move**, not move-everything. Most chorus-in-gathering surfaces are *regenerated* by the owl-api fan-out and **ripped in place** — never moved. Only *residue* (no generated successor — the convergence/ICD/NiFi vertical + genuinely-bespoke pages) is a real move, into its ADR-041 home, each move retiring its source under a retirement gate.

## Sequence (ADR-041 §6)

1. **ADR-041 lands** — the tree is decided before any fan-out leg writes a path (no hollow homes to ingest).
2. **Inventory-as-discriminator** (#2292) — classify every chorus-in-gathering piece `regenerated | residue`, tag each with its ADR-041 destination. Drafted from the 2026-06-12 3-agent sweep.
3. **Prep / decouple** (parallel, unblocked) — this card (#3097, chorus side) + #3383 (gathering-side NiFi). Close the couplings so moves are moves, not refactors.
4. **Moves** — residue only, attrition-filled; fan-out legs land generated chunks directly in new homes. Tail: #2298 (stray ADR-006), #2299 (final contract gate).

Driver: chorus code out of gathering **before** the crawler-rewrite leg, or the graph ingests hollow domains.
