Key Insight: The Enforcement Gap

Three types of enforcement exist. The gap between "described in a skill" and "enforced in a hook" is where inconsistent execution lives.

LevelMechanismCan Be Bypassed?Examples
Hard — Rust hookPreToolUse gate in chorus-hooks binaryNo — blocks the tool call at the harness levelTDD gate, search hierarchy, sensitive paths, accept gate, ICD gate
Hard — Skill gateStep in a skill that stops progressionYes — role can skip the skill entirelyDesign gate in /pull, smoke check in /demo, demo check in /acp
Soft — Skill textInstruction in skill markdownYes — role can ignoreWSJF priority, nudge exchange limit, autonomy rules
None — ConventionDescribed in CLAUDE.md or decisions but not in any gateYes — no enforcement at allBrief routing, handoff logging, cost awareness
