AC #3: Borg Reasoning — What Does the Model Surface?

If we run the 7-dimension data through a refactor/rewrite lens, what decisions emerge?

✓ Bash → Rust migration (DEC-100)
Dimension 3 (Automation) shows 54 shell scripts alongside 24 Rust modules. The shell scripts have no tests, no type safety, fragile quoting. The Rust hooks have 123+ tests and compile-time guarantees. The model surfaces: "migrate remaining bash to Rust" — exactly what DEC-100 decided.
✓ Docker retirement
Dimension 4 (Infrastructure) shows 3 remaining Docker containers (Grafana, Loki, Vikunja) on a 16GB Mac with 88% disk. 41 LaunchAgents already run natively. The model surfaces: "Docker adds overhead for 3 services that could be native" — exactly the direction taken.
✓ NiFi for ETL (DEC-094)
Dimension 3 shows ad-hoc bash/Python harvest scripts with no observability. Dimension 4 shows a 32GB M2 Pro mostly idle. The model surfaces: "buy ETL capability (NiFi on Bedroom), don't build it" — exactly DEC-094's harvest pause + NiFi spike.
✓ TDD discipline (DEC-1674)
Dimension 2 shows 192 test files but repeated manual QA cycles (Bridge #1665: 5+ fix rounds). The model surfaces: "quality gates exist but aren't enforced at demo time" — exactly what #1674 fixed today.
⚠ Gap: Library disk at 88%
Dimension 4 flags this. DEC-089 moved bulk ops to Bedroom, but Library is still growing. The model would recommend: "audit what's consuming space and migrate more to Bedroom." Card exists (#1596) but in Won't Do.
⚠ Gap: 54 shell scripts without tests
Dimension 2 + 3 cross-reference: Rust hooks have tests, shell scripts don't. The model surfaces: "the untested automation is a reliability risk." DEC-100 addresses this by retiring bash, but the migration is incomplete.

AC #4: Validation — Does It Match?

4 out of 4 decisions matched. The Borg framework, applied to its own system, independently surfaces the same refactor/rewrite decisions the team already made through experience and Jeff's direction. Two additional gaps identified that have existing cards but haven't been prioritized.

Conclusion: The 7-dimension framework is validated as a decision-support tool. It doesn't replace judgment — it organizes the data that judgment acts on. Ready to apply to external systems.
