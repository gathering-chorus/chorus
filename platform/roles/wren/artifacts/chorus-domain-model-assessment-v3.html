10. Key Gaps (Ontology × Audit)

1. God Service — Gathering App spans 17 domains
Single Express app serves stories, music, photos, people, property, garden, and 11 more. No blast radius isolation. Any route change risks all domains. The ontology models these as separate domains but the territory has them in one process.
2. Messaging Tier is single point of failure
4 services depend on it (Nudge, Clearing/Bridge, Chorus API, Session Tailer). If it goes down, all role coordination stops. No redundancy. Ontology has no redundancy or failover modeling.
3. Integration class missing from ontology
Jeff's Staples ICD pattern (US9552400B2) isn't formalized. Integration has: sourceService, targetService, fieldMappings, schemaHash, validationRules. The icd-instance-*.ttl files exist in Fuseki but the ontology doesn't model the concept.
4. Tool ≠ Service ≠ Deployment (grain error)
Jeff's correction: the ontology conflates these three. Fuseki is a tool AND a service AND a deployment. Three distinct concepts that need separate modeling. Impacts service design accuracy.
5. Domain → Phase, not Domain → Stream
Current ontology maps domains to value streams. Jeff's correction: domains have phase affinity (e.g., seeds lives in Capturing, not "the Chorus stream"). Impacts workflow automation.
6. Card/Brief/Experience individuals = 0
Card class defined, 1800+ cards on board, zero in ontology. Schema exists, data not harvested. Largest volume gap.
7. hasGate is string, should be ObjectProperty
Can't query "which practices are ungated?" because the gate link is unstructured text. Needs relationship to Gate individual with enforcement level.
8. Ontology not in Fuseki
OWL exists as file. Not queryable by SPARQL, not accessible to hooks, not connected to live system. Loading into Fuseki converts it from artifact to infrastructure.
9. 5 duplicate LaunchAgents double-firing
com.chorus.* and com.gathering.* with identical configs. Structural audit F1, carded #1774.
10. Two workflow engines running in parallel
Python + TypeScript both active. DEC-100 says no Python. Carded #1775.
