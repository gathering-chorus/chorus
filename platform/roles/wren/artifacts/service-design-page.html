5 Test Contract

Derived from the experience statement. If the test doesn't match the experience, the test is wrong.

  • Unit: SMS adapter parses plain URL, rich link (Apple Music), photo, hashtag-only
  • Unit: Seed handler correlates hashtag across separate webhook calls (both orderings)
  • Integration: POST webhook with Apple Music payload → POST webhook with #wren → assert brief file exists in destination
  • Integration: POST webhook with photo payload → POST webhook with #wren → assert brief file exists with media attached
  • Integration: Reverse order (hashtag first, content second) → assert brief still arrives
  • Synthetic: Hourly cron sends test payload through full pipeline, verifies brief arrival

6 Dependencies

7 Design Gate

All items checked before any card moves to Building.

  • Experience statement written
  • Integration path mapped with spine events
  • Key design constraints documented
  • Non-functionals specified (idempotency, observability, error visibility, monitoring)
  • Test contract derived from experience statement
  • Dependencies documented (upstream, downstream, shared state)
  • Language and data store chosen per DEC-100

The Pattern: Define → Map → Build

Same discipline as ICD governance (DEC-095), convergence practice, and Jeff's Staples integration architecture. No code without a contract. No building without a design artifact.
