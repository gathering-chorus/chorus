# 2026-03-11 — Two-hub restructure, nav-tree maturity

Big session. The nav-tree.json went from a brittle build artifact to a live-reloading config file (#1287), then the entire information architecture got restructured (#1294). System branch eliminated — its docs redistributed between Gathering (product vision) and Chorus (team method). The mind map went from 7 spokes to 6, with Gathering at center.

The recurring pattern: competing toggle systems for Self inner-ring vs generic sub-nodes. Patched the desync but the real fix (#1291) is unifying them. Two state machines controlling one visual element is always going to drift.

Jeff's "just make it show here" moments are the best design signals — he points at the screen and the answer is obvious. The Buddha photo on Practicing, the About node for product docs. No spec needed. The hot-reload made this possible: edit JSON, refresh, iterate. That's the workflow nav-tree.json was always meant to have.

The CSS opacity fix (rgba → rgb) was a good reminder: transparency in SVG-heavy UIs creates visual noise that's hard to diagnose. Solid backgrounds are almost always the right call when nodes overlap lines.
