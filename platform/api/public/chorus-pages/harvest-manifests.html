Sources
Raw files on disk or network
What
Original media libraries, backups, and exports scattered across machines and drives.
How
Inventory via SSH, mount checks, XML discovery.
Why
Can't harvest what you can't find — this is the starting inventory of everything that exists.
→
Stage
rsync to staging area
What
Copy source files to a centralized staging area on Bedroom Mac.
How
rsync over SMB/SSH to /Volumes/VideosNew/Gathering/Music/ToDo/.
Why
Sources live on different machines and drives. Staging consolidates them so the pipeline operates on a single path without flaky mounts.
→
Extract
Parse XML/files to tracks
What
Parse source files (iTunes XML, file trees) into a flat list of track metadata.
How
XML parser reads iTunes Library.xml → one JSONL record per track with artist, album, duration, play count, genre.
Why
Raw sources are proprietary formats. Extract normalizes them into a common structure the pipeline can consume.
→
Transform
Dedup, group into albums + TTL
What
Deduplicate tracks, group into albums, and generate RDF (Turtle) files.
How
Hash-based dedup on artist+title+duration → album grouping → one .ttl per album with tracks, artists, genres, artwork.
Why
Fuseki stores RDF. This converts flat track lists into linked data — albums connect to artists, genres, and artwork.
→
Reconcile
Diff against canonical, import unique
What
Diff secondary source tracks against the canonical Apple Music library. Import unique files.
How
Compare by artist+title+duration hash. Verify MP3s on disk. Stream-import unique tracks into Apple Music via osascript.
Why
Multiple sources overlap. Reconcile identifies what's genuinely new and merges it into the canonical library so Navidrome serves everything from one place.
→
Load
Sync TTL to Fuseki graphs
What
Upload generated TTL files to Apache Fuseki as named graphs.
How
HTTP PUT each .ttl to Fuseki /pods dataset. One named graph per album. Parallel uploads (2-8 workers).
Why
Fuseki is the SPARQL triplestore powering search, browse, and cross-domain queries. Loading makes data queryable.
→
Fuseki
RDF knowledge graph
What
Apache Jena Fuseki — the RDF knowledge graph storing all harvested data.
How
SPARQL queries serve album browse, artist search, cross-domain links. Named graphs = one per album/collection.
Why
RDF lets domains link — a song connects to an artist, a photo to a location, a story to a value. Fuseki is the single query layer across all of Jeff's data.
→
Deprecate
Archive staging, reclaim disk
What
Archive or delete the staging copy in ToDo after Fuseki load is verified.
How
Verify Fuseki counts match expected. Move ToDo/<source> to Done folder or delete with explicit flag. Mark deprecated in manifest.
Why
Staging data is a copy — once verified in Fuseki, keeping it wastes disk. Deprecation completes a source's lifecycle.
