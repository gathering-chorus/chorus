Principles — Reference Implementation

The first substrate-class domain put through the full treatment — graph as source, API as write path, drift test as gate, cite-by-ID, hook as backstop. Captured fresh so the shape transfers to Practices, Policies, Decisions, ADRs, Services.
Cards: 14 · Shipped: 5 · Queued: 9 · Rendered:

The Heart

This page exists because the principles arc was the first time we put a substrate-class domain through the full treatment. The shape that emerged is reusable. Five moves, in order:

  1. Graph as the single source. One named graph holds every instance; nothing is paraphrased anywhere else.
  2. API as the only write path. POST/PUT/DELETE through Athena; hand-edits to TTL are blocked at commit.
  3. Drift test as the gate. A static baseline (book/principles-reconstructed.html) is held against the graph; divergence fails CI.
  4. Cite-by-ID, never inline. Role files reference principle URIs and emphasize a subset; the lens is editorial, the body is canonical.
  5. Hook backstops the affordance. The typed API becomes the path of least resistance; the pre-commit hook bites the path it replaces.

This shape transfers. Practices, Policies, Decisions, ADRs, Services — every substrate-class domain that wants to stop being documentation and start being a contract goes through these five moves. The principles arc is the proof; everything past #2469 is replication.

How we got here

#2447 found the gaps — the live page was missing parents and the graph had no drift test. #2314 moved the substrate — 27 instances out of the ontology graph into the instances graph, an Athena write API, and the Loom GET retired to a 308 so no parallel implementation could grow back. #2157 proved the API by writing a meta-principle ("tests hermetic by default") through it as the first content write. #2473 populated 18 missing team principles through the same path — Jeff stamped wording in three batches, no TTL touched. #2449 closed the loop: role principles.md fragments became citation-only, drift-proof by construction. The arc took five cards because each one removed one path back to the old way.

What we deliberately didn't do

  • Auto-update role files when a principle changes. Emphasis is editorial — the role chooses what to weight. Auto-sync would erase that judgment. Citations stay live; emphasis stays authored.
  • Generate this page. Jeff is reading-as-thinking right now; a static surface he can mark up beats a dashboard. Generation comes after the shape settles.
  • Allow Loom GET as a parallel read path. The 308 is non-negotiable — two read shapes for the same data is how drift starts. Retirement was part of the contract, not a follow-on.
  • SHACL across graphs in #2314. The boundary check (label-required → 400) covers the immediate need; cross-graph SHACL waits for #2469 where it can be tested hermetically. Don't ship enforcement you can't test.
  • Update/Delete MCP tools in #2476. Create has unproven cadence; PUT/DELETE on REST stay available. Adding tools nobody reaches for is anti-affordance.

Key Invariants

The non-obvious rules that thread through multiple layers. Each is a single sentence with a one-line rationale.

I-1The graph is the source of truth. Every other surface — page render, role citations, MCP tools, boot envelope — derives from the graph. No surface caches. No surface paraphrases. Drift starts the moment a copy lives somewhere else. Scar: role principles.md files used to inline principle text; every wording tweak meant a three-place edit Jeff or the role would forget. #2449 retired the inlining for exactly this reason.
I-2URIs are stable across all migrations. Citations, links, and external references stay valid forever. Renaming a graph location does not rename the subject. Established by ADR-025 §Decision; assumed by every consumer.
I-3Schema and content occupy different graphs. Schema in urn:chorus:ontology, content in urn:chorus:instances. SHACL validates across the boundary. Mixing them is the original sin #2314 was filed to undo.
I-4One source, multiple call shapes. REST, MCP, SPARQL, page render — all delegate to the same Athena handler reading the same graph. No competing implementations. New transports are surfaces over the source, not new sources. Scar: the Loom GET was about to become a permanent parallel read path. #2314 retired it to a 308 in the same commit that introduced the canonical Athena route — closing the door before a second implementation could harden.
I-5Single-verb-single-target tool granularity. Each operation gets its own tool: chorus_principles_list, chorus_principle_get, chorus_principle_create. No chorus_principles_op(action,…) collapses. Granularity is the affordance — Claude reaches for the friendliest tool, and the friendliest tool is the most specific.
I-6Tool descriptions are first-class doc strings. Read every invocation, not once at boot — 100x cadence vs CLAUDE.md. Anti-patterns belong in the description string, not in a brief. Descriptions are governed artifacts.
I-7Rule expression beats rule paraphrase. Don't restate principles; cite IDs. Don't reword principles in role files; reference them. The role file is a lens, not a copy. grep -c '^[0-9]\\.' on a principles fragment must return 0. Scar: three role files, three slightly-different paraphrases of the same principle, no way to tell which was current. The cite-by-ID rule exists because we lived with the alternative.
I-11Every principle is read three ways; the third is yours. rdfs:comment is the permaculture source. chorus:techReading translates it to engineering. chorus:jeffReading is Jeff's reading — what the principle means in this portfolio, on this team. The third layer is what makes a principle his and not paraphrased Hemenway. Without it, this is a textbook; with it, it's a contract he signed.
I-8Counts grow; tests assert floors. Hard-coded counts (12 edges, 27 instances) become stale the moment Jeff adds a principle. ≥N assertions stay valid forever.
I-9Hooks bite where the wrong path is tempting. The pre-commit hook on chorus.ttl exists because hand-editing the TTL was the path of least resistance before #2314. Now the typed API is the path of least resistance. The hook backstops the affordance.
I-10Atomic per class, not atomic across classes. Each migration child (Principle, Practice, Policy, …) commits in one shot — data move + handler flip + test update. Cross-class atomicity is sacrificed for testability and reversibility. Order is lowest-blast-radius first.
