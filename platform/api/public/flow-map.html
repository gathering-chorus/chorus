Page Render Flow

HTTP Request → Express → SPARQL → EJS → HTML Response

1
HTTP Request → Router
GET /path + query params
Express routing table
2
Middleware Stack
Auth + CSRF + CSP nonce + visibility rules
auth, csp-nonce, collection-visibility middleware
3
Handler Execution
Construct SPARQL query from route params
Handler class (MusicHandler, BookHandler, etc.)
4
Cache Check
Query hash → 60s TTL in-memory cache
SparqlService cache (hit/miss tracked)
5
SPARQL Query → Fuseki
URL-encoded query to /pods/query endpoint
App container → Fuseki container (port 3030)
6
Fuseki Response
JSON bindings: { head: { vars }, results: { bindings } }
Fuseki SPARQL endpoint
7
Handler Render → EJS
res.render(template, { results, locals })
Express/EJS engine
8
Partial Inclusion
head, navbar, page-head, content-actions, footer, page-close
EJS include directives — shared partials
9
HTML Response → Browser
Rendered HTML + inline CSS/JS with CSP nonce
Container → Client

Route Decision
#hashtag → collection (idea→ideas, garden→garden-bed, etc.) Default: seeds
HASHTAG_ROUTES lookup in SeedHandler
4
OG Metadata Fetch (if link)
URL → title, description, image from meta tags
App → External webpage
5
Pod Write

6
Fuseki Sync
New triples POST to /pods dataset
App → Fuseki (async)
7
Twilio Response

App → Twilio
8
Page Renders Seed
SPARQL query returns new seed on /seeds page
Page render flow (see above)

Nudge Pipeline

Event → Queue or Inject → Drain on idle → 2-exchange limit

1
Trigger Event
Demo ready, card mentioned, brief sent, or role-to-role tap
nudge.sh send <role> "message"
2
Queue or Inject
Target idle → inject immediately. Target busy → queue to /tmp/nudge-inbox-<role>/
nudge.sh (checks role-state)
3
Stale Discard (>24h)
Auto-drain discards messages older than 24h with spine event
role-state.sh drain trigger (#1224)
4
Drain on Idle/Session Start
role-state.sh waiting/idle → drain inbox. werk-init.sh → drain at boot.
Auto-drain (#1224)
5
2-Exchange Limit
3rd nudge between same pair refused → redirect to /clearing
nudge.sh counter (DEC-079)

Gemba Flow

/gemba → Tail builder → Cron loop → Commentary → Jeff observes → Exit

1
/gemba <role> Invoked
Card context (board-ts view) + tail snapshot (last 20 lines) + declare observing
Observer role — 3 calls, <5 seconds (DEC-071)
2
Background Tail Starts
chorus-query.sh tail <role> --follow (background task)
Observer → Chorus query service
3
Cron Loop Fires
Every 1 minute: check tail for new batches, digest for Jeff
CronCreate (self-sustaining loop)
4
Play-by-Play Commentary
[HH:MM] what happened · what it means · flag if any
Observer interprets, doesn't dump raw output
5
Jeff Interacts
Comments, questions, feedback — relayed or carded, not fixed in-flight
Jeff → Observer (ceremony discipline)
6
Exit (TTL or Signal)
10-min TTL, "stop", card.accepted/rejected, or Jeff redirects
CronDelete + TaskStop + debrief
