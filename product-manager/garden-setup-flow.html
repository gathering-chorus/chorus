Steps 1-3: One-time setup (anytime)

Setup Phase

One-time property and garden structure definition

1
Define Your Property
Establish the land — orientation, sun, soil
Ontology Only
What Jeff Does
Answers basic questions: address, lot orientation (Jeff's faces east/south), sun exposure patterns across the day.
Data Produced
jb:Property instance, jb:Land with aspects (sun, soil, drainage). RDF triples in Fuseki.
UX Sketch
Simple form: address, compass orientation picker, lot size estimate. Could be a one-time wizard.
Observe and Interact
Ontology exists (#1406). No UI yet.
2
Draw Your Zones
Map the spatial layout around the house
Built
What Jeff Does
Hand-draws the zone map on paper, photographs it, or draws directly on screen. Jeff drew zones on a printed screenshot and sent as SMS seed.
Data Produced
jb:Zone instances with direction, sun exposure, relative position. 11 zones from Jeff's hand-drawn spec.
UX Sketch
SVG canvas with house at center, click to add/name zones. OR upload photo of hand-drawn map and trace zones over it.
Connect (Relative Location)
Zone map built (#1341). Interactive editing not yet built.
3
Add Beds to Zones
Define individual planting areas within each zone
Ontology Only
What Jeff Does
Within each zone, defines individual garden beds: raised bed, ground plot, container, etc. Names them, sets dimensions.
Data Produced
jb:GardenBed instances linked to zones via jb:hasBed. Type (raised/ground/container), dimensions.
UX Sketch
Click a zone on the map, add beds within it. Name, type picker, optional size. Visual placement on zone canvas.
Use Small-Scale, Intensive Systems
GardenBed class exists. No UI for adding beds.
Seasonal Cycle Begins
4
Plant Your Garden
Record what goes where and when
Not Built
What Jeff Does
Records what's planted where and when. Selects from known plants or adds new ones. Companion planting notes.
Data Produced
jb:Planting instances: which jb:Plant in which jb:GardenBed during which jb:Season + year.
UX Sketch
Select a bed, add plantings. Auto-suggest from PictureThis inventory. Season auto-detected from date.
Each Element Performs Multiple Functions
Planting/Plant/Season classes exist. No planting UI.
5
Track Through the Season
Observe, photograph, note what's growing
Not Built
What Jeff Does
Observes, photographs, takes notes. What's growing, what's struggling, what to change. Daily or weekly check-ins.
Data Produced
Observations linked to plantings: photos, notes, health status (thriving / struggling / dead).
UX Sketch
Tap a bed, see current plantings, add observation (photo, note, health). This is where the map becomes alive.
Catch and Store Energy
Not built. The daily/weekly use case.
6
Harvest and Reflect
End of season review — what worked, what didn't
Carded
What Jeff Does
End of season: reviews each zone. What worked, what didn't, what to change next year. Captures lessons.
Data Produced
Seasonal summary per zone/bed. Year-over-year comparison data. Linked to the Reflecting spoke.
UX Sketch
Season ends: prompted to review each zone. Visual diff: this year vs last year. Cards generated for changes.
Mistakes are Tools for Learning
#1343 (seasonal toggle) carded but not built.
7
Plan Next Season
Adjust, move, try new things based on reflection
Not Built
What Jeff Does
Based on reflections, adjusts the plan. Moves plants between beds, tries new varieties, changes bed layouts.
Data Produced
New jb:Planting instances for upcoming season, linked to previous season's reflections.
UX Sketch
Copy last season's layout as starting point, modify. Drag plants between beds. The full cycle: Sowing → Growing → Harvesting → Reflecting → Sowing.
Collaborate with Succession
Not built. Full cycle completion.
