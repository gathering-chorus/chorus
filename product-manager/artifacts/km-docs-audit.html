The Problem

55 docs in the navbar. 140 in doc-catalog. 28 files exist in both docs/ and data/about/ with no clear canonical. 7 of those have significant drift — API.md is 63% different between copies. Jeff navigates the navbar to find docs, but 30 of 55 navbar entries are architecture/engineering reference that only the team uses.

Recommendation Summary

KEEP in navbar (~40): All collection pages (Photos, Music, People, etc.), dashboards (/jeff, /werk, /loom, /cost), tools (/search, /seeds), plus key product docs (Vision, Glossary, Persona, Taxonomy, Interaction Patterns, Cheat Sheet).
MOVE to doc-catalog only (~30): All 14 ADRs, all architecture docs (C4, Living Architecture, Conceptual Model, etc.), all engineering refs (API, Testing, Style Guide), all operations docs (Infrastructure, Startup Process), research docs (Heidegger, Homeostasis, SOLID Pods).
DELETE stale copies (7 files in docs/): API.md (63% drift), SOLID-AUTHENTICATION.md (50%), DECISIONS.md (52%), GATHERING_VISION.md (40%), CONCEPTUAL_MODEL.md (29%), ARCHITECTURE_DECISIONS.md (25%), SITE_MAP.md (13%).
INVESTIGATE before deleting (2): DECISIONS.md — docs/ is 73KB vs data/about/ 35KB (docs/ may be newer). SOLID-AUTHENTICATION.md — docs/ is 13KB vs 6.5KB.
