DNA — What Persists

These survive session boundaries, role reboots, and context loss. They are the team's behavioral genome.

Decisions (109)

  • Locked policies — DEC-107 (nudge), DEC-048 (demo gate), DEC-100 (no bash APIs)
  • Stored in decisions.md — versioned, searchable
  • Hooks enforce them — the decision IS the code

Gates (33 modules)

  • Rust binary — compiled, deployed, persistent
  • PreToolUse + PostToolUse on every action
  • TDD, synthesis, blast radius, ICD, card type
  • Circuit breaker — self-healing on failure

Skills (10+)

  • /pull, /demo, /acp, /pair, /gemba, /clearing, /reboot
  • Behavioral protocols — the same steps every time
  • Symlinked from chorus repo — source-controlled

Card Types (5)

  • new, enhance, fix, chore, swat
  • Drive gate behavior — fix gets git log + Loki
  • Drive signal level — swat = critical, new = info
  • Drive interaction pattern — fix = SWAT, new = ideation

Pulse

  • Heartbeat (periodic) + nerve impulse (event-driven)
  • chorus-log.sh --level (info/warn/critical)
  • Spine events — the nervous system
  • 9,714 events/day, classified by urgency

Expression — What You See

These are the surfaces the DNA produces. Click to drill into detail. Each surface consumes DNA components.

The Clearing (:3470)

  • Four role tiles — card, state, last action
  • Card detail on click — AC, blast radius, domain
  • Ambient pulse events — card lifecycle visible
  • Consumes: Pulse, Card Types, Gates (circuit break alerts)

/werk

  • Value stream stages — Directing → Designing → Building → Proving
  • Spine event timeline
  • Instruments and metrics
  • Consumes: Pulse (spine events), Decisions (stage definitions)

/loom

  • Team ownership — who does what
  • Role metrics — build:clean ratio
  • Attention contract status
  • Consumes: Decisions (DEC-030 ownership), Skills (role protocols)

/flow

  • Board state — cards by status
  • Domain sections with aging flags
  • Fix:feature ratio from card types
  • Consumes: Card Types, Pulse (card events)

Doc Catalog

  • 150+ HTML artifacts — searchable, filterable
  • Three filter dimensions — Werk stage, life loop, domain
  • Design artifacts linked to cards (DEC-109)
  • Consumes: Decisions (DEC-109 linking), domain structure

Memory (Chorus Search)

  • 97K messages indexed — FTS + semantic
  • Decisions, stories, session history
  • Long-term awareness layer
  • Consumes: all DNA (everything searchable)

Borg

  • Pattern recognition across sessions
  • Stall detection, flow metrics, attention gaps
  • The brain interpreting the pulse
  • Consumes: Pulse (raw signal), Card Types (fix ratio), Decisions (thresholds)

Five Layers Across Both Rings

LayerDNA (persists)Expression (visible)
TimeCard lifecycle stages, session boundaries/werk timeline, aging flags on /flow
Pulsechorus-log.sh --level, spine events schemaClearing ambient events, team-scan injection
MemoryDecisions log, stories, session transcriptsChorus search, /chorus memory node
KnowledgeCard types, domain structure, ICD contracts/loom roles, doc-catalog, domain pages
ContextGates (what's allowed), skills (how to proceed)/flow board state, Clearing role tiles
The key insight: DNA without Expression is invisible — decisions nobody reads. Expression without DNA is ephemeral — surfaces that reset every session. The genome model is both: persistent behavioral code that produces visible, usable surfaces.

Forkability

Another team forks the Chorus repo. They get:

  • DNA: 109 decisions, 33 gate modules, 10+ skills, 5 card types, pulse schema
  • Expression: Clearing UI, /werk, /loom, /flow, doc-catalog, Borg
  • They mutate: Their own decisions, their own gate thresholds, their own skills. Same species, different expression.
