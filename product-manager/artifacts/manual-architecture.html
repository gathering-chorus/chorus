4. Harvest System

Pipeline pattern: Source → Extract → Transform (normalize URIs, build triples) → Load (Turtle + Fuseki) → Verify (count, check graphs) → Manifest (record state).

Each domain has its own pipeline. Harvest is continuous, not one-time. Pipelines detect changes and update graph.

10. Architectural Principles

1. Ontology is the architecture. RDF model isn't a data layer under the app. It IS the app. Extend system = extend model.
2. Privacy is structural, not policy. Self on physically separate machine. Trust boundary is network cable.
3. Bind-mount everything you can. Views, static files, pod data. Changes live without deploy. 10x productivity multiplier.
4. Two machines are the cloud. No external deps for core function.
5. Observe everything, alert selectively. Metrics cheap to collect. Ops agents filter signal.
6. The method is the product. Every process improvement = product improvement.
