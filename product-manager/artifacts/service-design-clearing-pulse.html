Owner: Silas (infra) + Kade (UI) • Language: TypeScript • Port: 3470 • Domain: coordination

1. Experience

I open one URL and see what everyone is doing. I type and they respond. I watch and talk on the same screen.

One screen. Jeff sees the system state (tiles, messages, session streams, board flow) and can talk to any role — all without switching tabs, windows, or URLs. Dashboard by default. Chat when he wants it. The Clearing is the room where Jeff and the team are always present.

4. Non-Functionals

AvailabilityAlways-on. LaunchAgent restarts on crash. If The Clearing is down, Jeff is blind to the system.
LatencyTile refresh < 5s. Session tail < 2s. Chat responses: Anthropic API latency (model-dependent).
AuthLocal/LAN pass. Remote (Cloudflare tunnel) requires token. Token in /tmp/bridge-auth-token.
ObservabilityThe Clearing IS the observability surface. Log to /tmp/chorus-bridge.stderr.log. Health: GET /health.
Chat stateTranscripts auto-saved every 30s. Indexed into Chorus FTS5. Decisions auto-extracted to briefs.

5. Failure Modes

Jeff must NEVER see: Stale tiles showing yesterday's state while roles are building. If polling fails, show staleness visibly (red border, "stale" label).
Jeff must NEVER see: Session stream gap — role is active but stream shows nothing. WebSocket must auto-reconnect and show the gap.
Jeff must NEVER see: Clearing decisions lost — DECISION: prefix lines not captured. Governance artifacts.
Jeff must NEVER see: Message loss — he types something and it disappears. Persist to rolling buffer immediately.
System handles: Crash recovery — LaunchAgent restarts, WebSocket clients reconnect within 5 seconds, tiles repopulate from live data.
System handles: Chat session end — transcript saved, decisions extracted, indexed. Clean close.
