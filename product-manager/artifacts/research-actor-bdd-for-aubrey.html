What Jeff Built Today

A plain-language explanation for Aubrey • April 1, 2026

The Short Version

Jeff manages a team of three AI assistants that help him build software. Today, he figured out a way to tell them exactly how to behave — not by writing long instructions they forget, but by drawing simple diagrams that automatically turn into tests the system runs every time.

Think of it like this: instead of telling your employees "be nice to customers" and hoping they remember, you draw a flowchart of exactly what to do when a customer walks in, then install a camera system that checks whether they followed the flowchart.

Why This Matters

Jeff has given his AI team the same feedback over 120 times. "Don't ask me questions before investigating." "When I say stop, stop." "Don't apologize — just change the behavior." Every session, the AIs start fresh and make the same mistakes because they don't have structural memory of how Jeff wants to be treated.

Today he turned those 120 corrections into something permanent.

How It Works

Imagine you're training a new restaurant staff. You could write a 50-page employee handbook that nobody reads. Or you could draw a simple picture: "Customer walks in → greet within 10 seconds → seat at open table → bring water before they ask." Then you test it: send in a fake customer, see if the staff follows the picture. If they don't, the system tells them what they missed before the real customer ever sees a problem.

That's what Jeff built today, except for AI agents instead of waitstaff:

Step 1: Draw the picture. Jeff described how each interaction should work. "When I report a problem, ask me one question about what I'm seeing, then go investigate on your own. Don't come back and ask me more questions — come back with answers." That became a simple actor flow diagram.

Step 2: Write the test. The diagram turned into test scenarios written in plain English:

Given Jeff says "seeds are broken"
Then the role asks one short question about symptoms
When Jeff describes the symptom
Then the role investigates using memory and logs
And the role does not ask Jeff another question before reporting findings

Step 3: The system checks itself. Every time the AI team starts working, these tests run automatically. 98 scenarios, 517 checks, 18 seconds. If any behavior drifts from Jeff's expectations, the test catches it before Jeff has to.

Why It's Interesting

There's an academic field that tried to solve this problem 25 years ago — a thing called Agent UML. Researchers drew diagrams for how software agents should talk to each other. It faded because the diagrams were just pictures. Nobody could test them. Nobody could enforce them.

Jeff's version does all three: the diagram describes the behavior, the test proves it works, and a runtime gate blocks the wrong behavior before it happens. Each piece existed separately. The connected pipeline — from Jeff's words to a running enforcement system in one afternoon — is new.

The discipline that would have broken a human team — spending time on careful diagrams before writing any code — is free for AI agents. They don't get impatient. They don't skip steps because they're eager to start coding. Jeff holds the frame ("draw the diagram first"), and the AIs follow the path. The ideas survive the translation from concept to code because the definition work doesn't drain the energy needed for building.
