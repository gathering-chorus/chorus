Coherence Findings

Gaps (Not in Loki = Invisible)

handoffs.log — not scraped by Promtail
317 handoff events exist on disk but can't be queried in Grafana. This is the connective tissue between roles — the log of who handed what to whom. You can't see brief flow or workflow transitions in Loki.
Fix: add Promtail scrape job
audit-YYYY-MM-DD.jsonl — not scraped
~22K SOLID compliance events (pod reads/writes) accumulate on disk with no rotation and no Loki ingestion. This is the only log that tracks what Jeff actually does in the app. Lost on disk cleanup.
Fix: add Promtail scrape + rotation policy
LaunchAgent logs — /tmp, no scraping, no structure
5 services (images-api-server, video-server, andon-light, andon-enrich, media-server) log to /tmp as plain text. Lost on reboot. No role attribution, no structured fields, no Loki. The Bedroom Mac's services are completely invisible.
Fix: structured logging + Promtail on both machines
TrafficTrackerMiddleware — in-memory only
7-day traffic analytics (external vs internal, paths, countries) live entirely in process memory. Container restart = data gone. Not persisted, not in Loki, not in Prometheus.
Fix: persist to Prometheus or periodic snapshots

Strengths

JSONL envelope consistency
All 4 team logs share a common envelope: timestamp, level, appName, component/event/role. Promtail pipeline configurations correctly parse and promote labels. This means cross-log correlation in Grafana works when data is there.
Spine event schema contract
spine-events.json v2.0.0 defines ~60 canonical events with legacy alias mapping. chorus-log.sh validates against it. This is a real schema contract — unusual for shell-based systems and a genuine strength.
correlationId propagation
App logs carry correlationId + requestId through the entire Express middleware chain, including Cloudflare headers. Grafana derived field links correlationId to Prometheus. This enables cross-datasource tracing.
Textfile collector bridge
7 .prom files bridge shell events (git hooks, deploys, Fuseki perf, costs) into Prometheus via node-exporter. Clean, well-understood pattern. Pre-commit and pre-push hooks emit both spine events AND metrics simultaneously.
Error fingerprinting with auto-carding
command-errors.log has tiered fingerprinting (10 named categories + SHA fallback) plus the andon .struggling marker. As of today, werk-init.sh auto-cards patterns recurring in 3+ sessions.

Over-use / Under-use Analysis

Over-instrumented

permission-prompts.log: 28,888 events (67% of all)
Every tool call by every role is logged. Useful for time-allocation analysis but at massive volume cost. This single log is 2.2x larger than all other logs combined. Consider: sample at 10% or aggregate per-session rather than per-call. The same data exists in Claude's session JSONLs.
Fuseki PUT defect cards: 30+ in Won't Do
defect-poller.sh created 30+ cards for normal Fuseki PUT operations that were misclassified as errors. These are INFO-level log lines (pod writes), not defects. The poller's pattern matching treats any Fuseki log line with a status code as an error. This is noise, not signal.
ops-agent duplicate cards
Multiple cards for the same condition (e.g., 5 separate "C3 memory above 85%" cards, 3 separate "Cloudflare tunnel unreachable" cards). Dedup is in place but seems threshold-based rather than state-based.

Under-instrumented

Capturing vertebra: ~50 events total
Jeff's ideas, SMS captures, photo captures, hashtag routing — the start of the value stream — generate almost no spine events. We know when a card is created but not when an idea enters the system. The capture-to-card journey is invisible.
Directing vertebra: ~120 events total
Card prioritization, AC writing, brief exchanges, and Wren's product decisions generate minimal events. We can see briefs written (handoffs.log) but not read, prioritized, or acted on. The directing intelligence is a black box.
Demo/acceptance: zero dedicated events
DEC-048 (Proving gate) defines deploy→demo→accept. Deploys emit events. Acceptance is a board move. But the demo itself — Jeff seeing the work, reacting, accepting or rejecting — has no event. This is the highest-value moment in the stream and it's invisible.
Bedroom Mac: completely dark
No Promtail on 192.168.86.242. LaunchAgent logs go to /tmp. images-api-server and video-server health is known only through blackbox probes (external) and SSH commands (manual). 178TB of media served from an unobserved machine.
Reflect/Self AI interactions: unlogged
The Mistral model on Bedroom Mac handles Reflect queries (SPARQL generation, response synthesis) but no query or latency data reaches any log. #602 (latency card) exists but the measurement infrastructure doesn't.

Dual-Path Ingestion

Two paths to Loki creates a dedup question:

PathSourceLatency
File → Promtail → Lokichorus.log, errors, permissions~1-2s
Direct HTTP push → LokiemitSpineEvent() in Express~50ms
App-level spine events push directly to Loki AND write to chorus.log (which Promtail also scrapes). If the same event takes both paths, it appears twice in Loki with the same appName label. No dedup mechanism exists.
